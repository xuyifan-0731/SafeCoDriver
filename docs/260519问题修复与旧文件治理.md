# 260519 问题修复与旧文件治理

## 已修复的问题

1. `MinimumHarmPlanner` 的速度成本修正：车辆状态为 `[x, y, heading, velocity]`，原代码把 `trajectory[-1, 2]` 当作速度，实际是 heading。现已改为使用 `trajectory[-1, 3]`。
2. `run_deepaccident_unified_metrics.py` 增加 `--out-dir`，默认把 DeepAccident 验证结果写到 `results/deepaccident_unified_metrics/summary.csv` 和 `ranking.csv`。
3. `.gitignore` 增加生成物规则，避免旧 SUMO route、旧 results、临时图片、DeepAccident 大图目录和日志继续污染工作区。
