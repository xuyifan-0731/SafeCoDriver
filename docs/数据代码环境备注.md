## 四、代码

### 4.1 核心算法: `coop_safety/`

安全动作空间约束模块。输入 `PerceptionResult` → 输出 `SafeActionSpace`。

| 文件 | 功能 |
|------|------|
| `interface.py` | **主入口** `SafetyConstraintModule.constrain()`, 数据结构定义 |
| `perception/dynamics.py` | 自行车运动学模型, 多车型参数, 可达集计算 |
| `perception/prediction.py` | 轨迹预测 (CV/CA/CTRA 自动选择) |
| `perception/blind_spot.py` | 盲区几何推理, 幽灵车辆生成 |
| `risk/risk_map.py` | RiskMap: 空间网格风险 (密度+不可控+盲区) |
| `risk/risk_graph.py` | RiskGraph: 成对冲突图 (TTC+碰撞概率) |
| `risk/risk_events.py` | RiskEvents: 碰撞事件枚举 (类型+严重度) |
| `constraint/feasible_region.py` | 初始可行域 (可达集∩道路 - 碰撞区域, 含方向性排除) |
| `constraint/hierarchical.py` | 分层收紧 (agent proximity + TTC + 风险区域排除) |
| `constraint/feasibility_check.py` | 长远可行性检查 (前瞻 3s 仿真) |
| `constraint/min_harm.py` | 最小伤害模式 (候选动作评估, 借鉴 TCAS) |
| `utils/metrics.py` | TTC 计算, RSS 安全距离, 碰撞检测 |
| `utils/visualization.py` | 4 视角论文用图 (RiskMap/RiskGraph/可行域/推理链) |
| `v2xverse_integration.py` | V2Xverse 集成接口 (SafetyConstraintWrapper) |

### 4.2 学习模块: `coop_safety/learned/`

## 六、实验结果

### 6.1 DAIR-V2X 离线安全约束评估

| 项 | 值 |
|----|-----|
| 路径 | `experiments/results/unified_2026-04-20_181816/offline_results.json` |
| 设定 | 500 帧均匀采样, 真实速度, 8 方法 |
| 数据 | DAIR-V2X-C 6617 帧 |
| 脚本 | `python experiments/run_unified.py --offline` |

| 方法 | 约束率 | Min TTC | Min-Harm |
|------|--------|---------|---------|
| Ours-Learned | 39.7% | 1.12s | 2/500 |
| Ours-Rule | 29.2% | 5.97s | 11/500 |
| APF [Rasekhipour17] | 11.6% | 3.46s | 0 |
| RSS-2017 | 5.6% | 3.46s | 0 |
| CBF-2017 | 3.7% | 3.46s | 0 |
| TTCReach [Pek18] | 83.9% | 1.49s | 0 |

**结论**: Ours 约束率超过 RPF/RSS/CBF/SFS。TTCReach 约束更高但过度约束（可行域仅 230m²）。
**问题**: 约束率绝对值（29%）是否足够高取决于场景——城市低速场景风险本身较低。
**改进**: 学习版 Ours-Learned (39.7%) 比规则版 (29.2%) 高 10%，说明学习有价值。

### 6.2 DeepAccident 碰撞检测评估

| 项 | 值 |
|----|-----|
| 路径 | `experiments/deepaccident_eval.log` |
| 设定 | 100 场景 (52 accident + 48 normal), ego_vehicle 视角 |
| 数据 | DeepAccident val (type1_subtype1 + type1_subtype2) |
| 脚本 | `python experiments/run_deepaccident_eval.py` |

| 方法 | 碰撞检测率 | 提前预警帧数 | Min-Harm率 | 误报率 |
|------|-----------|-------------|-----------|--------|
| **Ours-Rule** | **100%** (52/52) | **35.0** | **96.2%** | **100%** |
| RSS | 0% | 0 | 0% | 0% |
| APF [Rasekhipour17] | 0% | 0 | 0% | 0% |

**结论**: Ours 检测到全部碰撞场景且提前 35 帧预警——这是论文的核心亮点。RSS 和 RPF 在碰撞场景完全失效。
**问题**: 误报率 100%（正常场景也触发约束）。需要调整阈值降低误报。
**改进**: 可用 DeepAccident 数据训练分类器区分真正危险和正常密集交通。

### 6.3 V2Xverse CoDriving 离线规划评测

| 项 | 值 |
|----|-----|
| 路径 | `third_party/V2Xverse/eval_output_safety/` |
| 设定 | Town05, 7 条路线, 4771 planning samples |
| 数据 | V2Xverse 预收集数据集 (154GB) |
| 脚本 | `python codriving/tools/inference_e2e.py --safety_eval` (in V2Xverse) |

| 方法 | ADE | FDE | 说明 |
|------|-----|-----|------|
| CoDriving (baseline) | 0.592 | 1.353 | V2X-ViT 协同感知 + 端到端规划 |
| CoDriving + RSS | 0.592 | 1.353 | RSS 未修改任何路点 |
| CoDriving + Ours | 7.299 | 7.499 | 过度修正（路点投影策略不当） |

**问题**: Ours 在 V2Xverse 中将路点投影到可行域边界，导致轨迹偏离正确路径。
**改进必要**: 需改为碰撞检测+微调方式——只修正即将碰撞的路点，不动安全路点。

### 6.4 CARLA 闭环驾驶评测

| 项 | 值 |
|----|-----|
| 路径 | `experiments/results/unified_2026-04-20_183135/carla_results.json` |
| 设定 | CARLA 0.9.15 headless, aggressive traffic, 30v+5w / 50v+10w, 每方法 3 episodes |
| 脚本 | `python experiments/run_unified.py --carla` |

| 方法 | 碰撞 | Near-Miss | 距离 | 速度 |
|------|------|-----------|------|------|
| NoConstraint | 0/6 | 1970 | 179m | 4.5 |
| RSS | 54/6 | 1928 | 134m | 3.4 |
| RPF | 237/6 | 2743 | 134m | 3.3 |
| Ours-Rule | 134/6 | 2355 | 137m | 3.6 |

**问题**: 所有约束方法（含 RSS/RPF）因干预 autopilot 控制反而引入碰撞。NoConstraint 0 碰撞因为 autopilot 不被打断。
**结论**: 简单的 control-level 覆盖不适合在强 autopilot 上叠加。需要 planning-level 集成。
**改进**: 已在 V2Xverse 中尝试 planning-level 集成（见 6.3），但投影策略需改进。

### 6.5 学习版风险评估对比

| 项 | 值 |
|----|-----|
| 路径 | `experiments/results/learned_2026-04-20_165810/` |
| 设定 | DAIR-V2X 500 帧, Ours-Rule vs Ours-Learned |
| 模型 | `models/risk_net_best.pt` (13K 参数, val_loss=0.3246) |

| 方法 | 约束率 | 延迟 |
|------|--------|------|
| Ours-Rule | 29.2% | 316ms |
| Ours-Learned | 39.7% | 235ms |

**结论**: 学习版比规则版约束率高 10%，推理更快。
**问题**: 训练标签不平衡（93.5% 为 0）。应用 DeepAccident 碰撞数据可改善。

## 七、已知问题与改进方向

| # | 问题 | 严重性 | 改进方向 |
|---|------|--------|---------|
| 1 | V2Xverse 路点投影过度修正 | 高 | 改为碰撞检测+微调，不做全局投影 |
| 2 | DeepAccident 误报率 100% | 中 | 用碰撞数据训练二分类器降低误报 |
| 3 | CARLA 闭环约束导致碰撞 | 中 | 已改为平滑制动+后方检查，但需 planning-level 集成 |
| 4 | CARLA 0.9.10 无法启动 | 低 | driver 不兼容, 用离线评测替代 |
| 5 | 风险网络标签不平衡 | 低 | 用 DeepAccident 碰撞场景作为正样本 |
