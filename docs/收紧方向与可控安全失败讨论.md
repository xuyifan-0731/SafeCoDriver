# 收紧方向与可控安全失败的设计讨论

## 问题 1: 应该从 2.4 到 2.6 收紧，还是 2.6 到 2.4 收紧？

### 方案 A: 从 2.4 (RiskMap) → 2.5 (RiskGraph) → 2.6 (RiskEvents) 收紧

**含义**: 先基于空间风险排除大范围高风险区域，然后精细化考虑交互冲突，最后根据具体碰撞事件进一步排除。

**优势**:
- **Coarse-to-fine**: 先用廉价的空间计算排除大块区域，减少后续精细计算的搜索空间
- **计算效率**: RiskMap 是 O(cells)，RiskGraph 是 O(N²)，RiskEvents 基于 RiskGraph 结果——先粗后细减少总计算量
- **逻辑自然**: "先确定哪些区域危险 → 再看具体是谁构成威胁 → 最后预判会怎么碰"

**劣势**:
- 空间层可能过早排除了实际安全的区域（因为不考虑交互信息）

### 方案 B: 从 2.6 (RiskEvents) → 2.5 (RiskGraph) → 2.4 (RiskMap) 收紧

**含义**: 先排除最明确的碰撞事件区域，然后基于 TTC 收紧，最后用空间风险做最终裁剪。

**优势**:
- **精确优先**: 最确定的风险（具体碰撞事件）先处理，避免被空间层的模糊判断误导
- **保守性可控**: 每一步收紧都基于更精确的信息

**劣势**:
- RiskEvents 依赖 RiskGraph（需要先有冲突边才能枚举事件），所以从 2.6 开始在逻辑上不自洽

### 当前实现与结论

**当前实现用的是方案 A** (2.4→2.5→2.6 计算，但约束收紧顺序是 Events→TTC→Map)。

具体来说：
1. 风险**计算**顺序: RiskMap → RiskGraph → RiskEvents（因为后者依赖前者的结果）
2. 约束**收紧**顺序: RiskEvents区域排除 → TTC排除区 → 风险网格排除（先精确后粗糙）

这是**计算 coarse-to-fine + 约束 fine-to-coarse** 的混合策略，兼顾了计算效率和约束精度。

**消融实验验证**: Ours-RiskMapOnly（仅空间层）约束率只有 19.3%，而 Ours-Full（三层）达到 29.2%，证明仅用空间层严重不足。

## 问题 2: 3.6 是否可以参考航空安全领域的"可控安全失败"理念？

### 航空领域的 TCAS RA 系统

**TCAS (Traffic Collision Avoidance System)** 是安装在民航飞机上的碰撞回避系统。当两架飞机距离过近时，系统经历以下阶段：

1. **Traffic Advisory (TA)**: 预警，提醒飞行员注意（类似我们的 CONSERVATIVE 模式）
2. **Resolution Advisory (RA)**: 强制指令，系统自动生成爬升/下降指令（类似我们的 MINIMUM_HARM 模式）
3. **Coordinated RA**: 两架飞机的 TCAS 通信协调，确保一个爬升一个下降（不会做出相同的规避动作）

**关键设计原则**:
- **分层升级**: TA → RA → 增强 RA，逐步升级介入强度
- **最小偏离**: RA 指令是"最小安全偏离"——只要求飞行员调整垂直速率到安全值，不要求激烈机动
- **确定性优先**: 宁可保守也不冒险——如果不确定是否安全，就假设不安全
- **可预测性**: 系统行为必须可预测，飞行员知道系统会做什么

**参考来源**: ICAO Doc 9863 "Airborne Collision Avoidance System (ACAS) Manual", 2006

### "可控安全失败"的工程理念

来自 Nancy Leveson 的 "Engineering a Safer World" (MIT Press, 2011)：

- **安全约束不是"永远不发生事故"，而是"事故发生时后果可控"**
- 关键概念：
  - **Fail-safe**: 故障时系统进入安全状态（如电梯失电时刹车锁死）
  - **Fail-operational**: 故障时系统继续运行在降级模式
  - **Graceful degradation**: 性能逐步降低而非突然崩溃

### 算法借鉴

我们的 3.6（最小伤害模式）可以这样借鉴：

| TCAS/航空概念 | 我们的实现 | 借鉴方式 |
|--------------|-----------|---------|
| TA (Traffic Advisory) | CONSERVATIVE 模式 | 检测到风险但还有可行域 → 轻微减速，不急刹 |
| RA (Resolution Advisory) | MINIMUM_HARM 模式 | 无安全可行域 → 生成最小伤害动作 |
| 最小偏离原则 | 平滑制动 + 后方检查 | 不做剧烈机动，只做最小必要的减速 |
| 分层升级 | NORMAL → CONSERVATIVE → MINIMUM_HARM | 逐级升级介入强度 |
| Coordinated RA | 协同感知 + 考虑后车 | 约束时考虑周围车辆反应（不急刹导致追尾） |
| Graceful degradation | 逐级放松约束 | 约束太紧时逐级放松，而非直接进入最小伤害 |

### 当前实现已有的借鉴

1. **三级模式** (NORMAL → CONSERVATIVE → MINIMUM_HARM): 对应 TA → RA
2. **逐级放松**: 约束太紧时先放松排除比例，再放松 TTC 阈值
3. **最小伤害选择**: 评估多个候选动作，选 severity × probability 最小的

### 可以进一步改进的方向

1. **"Coordinated RA" 对应**：在协同感知中，如果能获取其他车辆的规划意图，可以协调避让方向（但当前 DAIR-V2X 数据不支持）
2. **预测性 TA**：在进入 MINIMUM_HARM 前提前发出 CONSERVATIVE 预警，给规划器更多反应时间
3. **后方感知约束**：CARLA 闭环中的碰撞问题（急刹导致追尾）正是缺少"Coordinated RA"思想——当前已通过后方检查部分解决
