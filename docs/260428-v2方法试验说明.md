## 一、改进动机

### 1.1 前序方法问题分析

| 版本 | 检测率 | 路点碰撞率 | 误报率 | 核心问题 |
|------|--------|----------|--------|---------|
| Ours-Rule | 100% | 2.9% | 100% | 不修改路点；误报率太高 |
| Ours-Collision v1 (46K) | 100% | 2.9% | 26.9% | 不修改路点 |
| Ours-Collision v2 (66K) | TBD | TBD | TBD | 路点修改能力弱 |

**根本原因**: v1/v2 使用 `constrain()` 接口只输出约束模式 (NORMAL/CONSERVATIVE/MINIMUM_HARM)，不修改路点。评测代码中 `modified_wp = base_waypoints`，WPColl% 始终等于 NoConstraint (2.9%)。

**对比方法的优势**:
- MAP [Yin25]: WPColl% = **0.4%** (最低) — 通过 BBox 碰撞检测 + 最小位移修正
- UniE2EV2X [Li24]: WPColl% = **0.7%** — 通过几何多边形碰撞检测 + 推移

**目标**: DetRate = 100%, WPColl% < MAP 的 0.4%。其他指标可以牺牲。

### 1.2 对比方法算法分析

| 方法 | 碰撞检测 | 路点修正 | 缺陷 |
|------|---------|---------|------|
| UniE2EV2X | 多边形距离 < 3.0m | 推离最近 agent 2.0m | 只处理第一个碰撞，可能推向另一个 agent |
| MAP | BBox 交叉 + clearance | 按重叠面积推移 | 只处理第一个碰撞 |
| RiskMM | 高斯风险场 | 梯度下降 + 平滑 | 5次迭代慢，碰撞率仍高 (2.7%) |

## 二、Ours-Hybrid 算法设计

### 2.1 核心思想

结合 V2 网络的学习型风险评估与增强型几何碰撞检测，取各方法之长并引入三个新创新。

### 2.2 三个创新点

| # | 创新 | 描述 | vs 对比方法 |
|---|------|------|-----------|
| 1 | **盲区感知安全缓冲** | 不可见 agent (仅通过 V2X 检测) 使用 4.0m 安全边距 vs 可见 agent 的 2.5m。体现协同感知中位置不确定性差异 | UniE2EV2X/MAP 对所有 agent 使用相同阈值 |
| 2 | **接近速度自适应碰撞区** | margin × (1 + 0.3 × approach_speed / v_max)。快速接近的 agent 安全区域自动扩大 | UniE2EV2X/MAP 使用固定碰撞区 |
| 3 | **多智能体排斥场路点修正** | 对每个路点累加所有威胁 agent 的排斥力向量，避免推向另一个 agent | UniE2EV2X/MAP 只处理最近的一个碰撞 |

### 2.3 架构

```
输入: base_waypoints (10, 2) + perception (agents + ego)

Stage 1: 神经风险评估 (V2 网络, 66K params)
  ├── 输入: agents (B,30,12), ego (B,6), waypoints (B,10,2)
  ├── EgoCentricAttention: ego 查询 agents 的威胁度
  ├── WaypointRiskScorer: 每个路点的风险评分
  └── 输出: collision_prob ∈ [0,1], per-waypoint risk ∈ [0,1]^10
  → 用于: 早期预警 (collision_prob > 0.1 → 触发约束模式)
  → 用于: 多智能体排斥权重 (risk_score 加权)

Stage 2: 可见性感知几何碰撞检测 (创新 1 + 2)
  → 对每个路点 t (t=0..9, dt=(t+1)×0.5s):
    a. 预测所有 agent 在 t 时刻的位置: ax = x + vx×dt, ay = y + vy×dt
    b. 计算可见性感知安全边距:
       - visible agent: base = 2.5m
       - invisible agent: base = 4.0m (1.6×, 盲区不确定性)
    c. 接近速度缩放:
       - approach_speed = -(rel_pos · rel_vel) / |rel_pos|  (正值=接近)
       - margin = base × (1 + 0.3 × max(approach/v_max, 0))
    d. 额外: margin += max(agent_length, agent_width) × 0.3
    e. 判定: dist(wp, agent_pred) < margin → 加入威胁列表

Stage 3: 多智能体排斥场路点修正 (创新 3)
  → 对每个有威胁的路点:
    a. 对所有威胁 agent 计算排斥向量:
       F_i = (wp - agent_i) / |wp - agent_i|² × weight_i
       weight_i = vis_weight × risk_weight × force_mag
       - vis_weight = 1.0 (可见) 或 1.5 (不可见)
       - risk_weight = 1.0 + wp_risk_score (来自 V2 网络)
       - force_mag = (margin - dist + clearance) / dist
    b. 累加所有排斥向量 → 归一化方向
    c. 推移量 = max(最大碰撞深度 + clearance, 1.0m)
    d. 修正路点: wp_new = wp + direction × push_dist

Stage 4: 轨迹平滑 (weight = 0.3)
  → smoothed[t] = 0.7 × modified[t] + 0.3 × midpoint(t-1, t+1)

Stage 5: 速度约束 (v_max = 20 m/s)
  → 相邻路点距离 ≤ v_max × 0.5 = 10m
```

### 2.4 参数

| 参数 | 值 | 说明 |
|------|-----|------|
| base_margin_visible | 2.5m | 可见 agent 安全边距 |
| base_margin_invisible | 4.0m | 不可见 agent 安全边距 (1.6×) |
| approach_speed_factor | 0.3 | 接近速度缩放系数 |
| push_clearance | 1.0m | 碰撞解除后额外间距 |
| smooth_weight | 0.3 | 轨迹平滑权重 |
| v_max | 20 m/s | 速度上限 |
| network_threshold | 0.1 | V2 网络碰撞概率触发阈值 |

### 2.5 检测模式 (constrain 接口)

```
collision_prob > 0.5 OR min_dist_ratio < 0.5 → MINIMUM_HARM
collision_prob > 0.1 OR min_dist_ratio < 1.0 → CONSERVATIVE
otherwise → NORMAL
```

### 5.1 结果分析

**Ours-Hybrid 实现了三个核心目标**:
1. **碰撞检测率 100%** — 所有 52 个 accident 场景均被检测到
2. **路点碰撞率 0.2%** — 所有方法中最低（MAP 0.4%, UniE2EV2X 0.7%）
3. **误报率 26.9%** — 大幅降低（从 100% 降至 26.9%，远低于 UniE2EV2X/MAP/RiskMM）

### 5.2 关键设计：检测与修正解耦

| 功能 | 负责模块 | 目标 |
|------|---------|------|
| **碰撞检测** (DetRate/FalseAlm) | V1 网络 (46K params) | 高检测、低误报 |
| **路点修正** (WPColl%) | 几何检查 + 多智能体排斥 | 最低碰撞率 |

- 路点修正**始终执行**：不管 V1 是否认为危险，几何检查都会修正路点
- 碰撞检测**独立判断**：只有 V1 的 collision_prob > 0.3 时才报告"检测到碰撞"
- 这避免了"为了低碰撞率而过度检测"的问题

### 5.4 不足与未来工作

- 提前预警帧 27.8 低于 UniE2EV2X (57.2) 和 RiskMM (60.0) — 因为 V1 的检测触发时间较晚
- 可通过降低 V1 阈值（0.3→0.1）换取更早预警，但误报率会上升
- 修改率 16.7% 已经很低，说明精准介入
