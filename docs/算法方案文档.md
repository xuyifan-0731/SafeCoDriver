# 算法方案文档：基于协同感知的安全动作空间约束（精简版）

## 术语表与参考文献

> 本节对文档中所有专业术语给出定义和学术出处。

### 基础概念

| 术语 | 定义 | 出处 |
|------|------|------|
| **协同感知 (Cooperative Perception)** | 多辆车或车与路侧设备通过 V2X 通信共享传感器数据，融合形成更全面的环境感知。单车盲区可被其他车辆传感器覆盖。 | Xu et al., "OPV2V," ICRA 2022 [1] |
| **V2X (Vehicle-to-Everything)** | 车辆与一切事物的通信总称，含 V2V（车对车）、V2I（车对路侧设施）。 | 3GPP TS 22.185 |
| **动作空间 (Action Space)** | 自动驾驶车辆在某时刻可执行的所有合法动作集合。本文中由可行区域多边形 + 运动学约束定义。 | Sutton & Barto, "RL: An Introduction," 2018 [2] |
| **TTC (Time-to-Collision)** | 碰撞时间。假设两目标保持当前运动状态到碰撞所需时间。TTC = \|相对距离\| / 接近速度。 | Hayward, Highway Research Record, 1972 [3] |
| **RSS (Responsibility Sensitive Safety)** | Mobileye 提出的形式化安全框架，定义纵向和横向安全距离公式。 | Shalev-Shwartz et al., arXiv:1708.06374, 2017 [4] |
| **CBF (Control Barrier Function)** | 基于李雅普诺夫理论的安全约束方法，通过屏障函数 h(x) 保证系统状态始终在安全集内。 | Ames et al., IEEE TAC, 2017 [5] |
| **可达集 (Reachable Set)** | 给定当前状态和控制约束，车辆未来可达的所有状态集合。 | Althoff et al., IEEE T-IV, 2016 [6] |
| **自行车运动学模型 (Bicycle Kinematic Model)** | 将四轮车简化为前后两轮的运动学模型。状态 (x,y,θ,v)，控制 (加速度a, 转向角δ)。 | Kong et al., IEEE IV, 2015 [7] |
| **CTRA (Constant Turn Rate and Acceleration)** | 等转弯率等加速度轨迹预测模型，适用于转弯车辆。 | Schubert et al., IEEE FUSION, 2008 [8] |
| **TCAS RA (Traffic Collision Avoidance System)** | 航空碰撞回避系统，当飞机距离过近时自动生成爬升/下降指令。本文最小伤害模式借鉴其优先级逻辑。 | ICAO Doc 9863, 2006 [9] |
| **可控安全失败 (Controlled Safe Failure)** | 工程安全理念：无法完全避免故障时，确保故障模式可预测、可控、伤害最小。源自航空/核工程。 | Leveson, "Engineering a Safer World," MIT Press, 2011 [10] |

## 1. 问题定义

### 输入
协同感知系统（V2X）的融合结果：自车状态、周围交通参与者（位置/速度/类型）、道路信息、可见性（盲区）。

### 输出
安全动作空间：可行区域多边形 + 运动学约束 + 运行模式（NORMAL / CONSERVATIVE / MINIMUM_HARM）+ 推理逻辑链。

### 核心理念
一个**独立的、可插拔的安全约束模块**。任何自驾算法（端到端或模块化）只需在输出的安全空间内决策即可。

---

## 2. 方法概述

### 三个核心贡献

| 贡献 | 内容 | 对应的缺陷 |
|------|------|-----------|
| **贡献 1: 三层一致风险评估 + 分层约束收紧** | RiskMap（空间）→ RiskGraph（交互）→ RiskEvents（事件）三层风险，对应三层约束从粗到细收紧 | RSS[4] 只有纵向/横向单层安全距离，CBF[5] 无多粒度风险表示 |
| **贡献 2: 长远可行性检查** | 约束不仅保证当前安全，还前瞻验证未来 N 秒是否仍有可行动作 | 现有方法[4][5]只保证瞬时安全，可能约束到死胡同 |
| **贡献 3: 最小伤害模式** | 不可避免碰撞时，选择伤害最小的动作（借鉴航空 TCAS[9] 和可控安全失败[10] 理念） | 现有方法在极端场景下直接失效（无安全动作 → 系统崩溃） |

**附加贡献**：独立安全接口设计，同一约束模块可无修改地叠加到不同自驾算法上。

## 3. 三层风险评估

### 3.1 感知预处理

**车辆动力学 (方案 2.1)**：根据车辆类型匹配自行车运动学模型[7]参数。

**盲区推理 (方案 2.2)**：计算遮挡几何阴影区域；协同感知减少盲区；剩余盲区保守假设存在幽灵车辆。

**轨迹预测 (方案 2.3)**：根据状态选择 CTRA[8]（转弯）/ 等加速度（加减速）/ 等速度（默认）。

### 3.2 RiskMap：空间区域风险 (方案 2.4)

将自车周围空间切割为网格（5m×5m），每格计算：

**Risk = 0.4 × 密度分 + 0.3 × 不可控分 + 0.3 × 盲区分**

- 密度分：当前+预测未来的交通参与者数量（时间折扣加权）
- 不可控分：行人、自行车等不可预测参与者
- 盲区分：与盲区的重叠比例

分级：HIGH (≥0.7) / MEDIUM (0.4-0.7) / LOW (<0.4)

### 3.3 RiskGraph：成对冲突分析 (方案 2.5)

构建交通参与者的冲突图（节点=参与者，边=冲突关系），每条边包含：
- **TTC[3]**：碰撞时间 = |相对距离| / 接近速度
- **碰撞概率**：P = 1/(1+exp(k·(d_min - r_eff)))，d_min 为轨迹最小距离
- **碰撞窗口**：轨迹距离<阈值的时间区间 [t_start, t_end]

**与 RiskMap 的关系**：RiskMap 是"哪个区域危险"（空间级），RiskGraph 是"谁和谁可能碰撞"（交互级）。消融实验证明两者不可互替。

### 3.4 RiskEvents：碰撞事件枚举 (方案 2.6)

将 RiskGraph 的抽象冲突实例化为具体场景，每个事件包含：
- 参与者 ID、碰撞类型（追尾/侧面/正面/交叉/行人）
- 严重程度 = 0.4×速度因素 + 0.3×质量因素 + 0.3×角度因素
- 空间区域多边形、时间窗口、概率

**与 RiskGraph 的关系**：RiskGraph 说"A 和 B 的 TTC=2.1s"（概率关系），RiskEvents 说"A 和 B 将在此处发生追尾，严重程度 0.7"（具体场景）。消融证明 RiskEvents 是**最关键的安全组件**。

## 4. 三层安全约束

### 4.1 初始可行域 (方案 3.1)

**可行域 = 动力学可达集[6] ∩ 道路边界**

通过自行车模型[7]采样不同控制输入，计算可达位置的凸包，再与道路多边形取交集。

### 4.2 分层收紧 (方案 3.2-3.4)

按**粗到细**顺序收紧（先大范围排除，再精细调整）：

**Step 1 — RiskEvents 收紧 (3.2)**：
可行域 -= 每个碰撞事件的空间区域（含概率加权缓冲）

**Step 2 — TTC 收紧 (3.3)**：
对 RiskGraph 中低 TTC 的冲突边，在碰撞点周围建排除区：
- TTC < 1.5s：大排除区（buffer = scale × (3.0 - TTC)）
- 1.5s < TTC < 3.0s：小排除区（buffer × 0.5）

**Step 3 — 风险区域排除 (3.4)**：
按 HIGH → MEDIUM 优先级排除高风险网格。每次排除后检查剩余面积，防止过度约束。

### 4.3 长远可行性检查 (方案 3.5, 贡献 2)

**核心思想**：当前安全 ≠ 未来安全。逼到死角虽然当前不碰撞，但未来无路可走。

**算法**：
1. 在当前可行域内采样多个控制动作
2. 用自行车模型[7]前向仿真 3 秒
3. 每 0.5 秒检查：自车是否仍在安全范围内？与预测智能体距离是否足够？
4. 至少一个动作通过完整前瞻 → 可行

借鉴 Pek & Althoff [13] 的 fail-safe 轨迹思想，但我们不需要精确的可达集计算，而是用采样仿真近似。

### 4.4 回退与最小伤害 (方案 3.6, 贡献 3)

当可行性检查失败：

**逐级回退放松**：
1. 降低风险区域排除比例（80%→50%→20%）
2. 提高 TTC 警告阈值（3.0s→4.0s→5.0s）
3. 减少碰撞事件排除范围

**全部失败 → 最小伤害模式**：
- 枚举候选动作（偏向紧急制动+转向规避）
- 评估每个动作的预期伤害：Cost = Σ(severity × probability × proximity × type_penalty)
- type_penalty：行人碰撞 ×5，正面碰撞 ×3
- 选择最小伤害动作

借鉴航空 TCAS RA[9] 的优先级逻辑和可控安全失败[10]理念。

## 5. 完整流程

```
协同感知结果 → 盲区推理 → 三层风险评估
                              │
                ┌──────────────┼──────────────┐
                ▼              ▼              ▼
            RiskMap      RiskGraph      RiskEvents
            (空间)        (交互)         (事件)
                └──────────────┼──────────────┘
                               ▼
                    初始可行域 (动力学∩道路)
                               │
                    ▼ Step1: -= RiskEvents 碰撞区域
                    ▼ Step2: -= TTC 排除区
                    ▼ Step3: -= 高/中风险网格
                               │
                    ▼ 长远可行性检查 (贡献2)
                    │     ├── 通过 → NORMAL 模式 → 输出
                    │     └── 未通过 ↓
                    ▼ 逐级放松重试
                    │     ├── 成功 → CONSERVATIVE 模式 → 输出
                    │     └── 全部失败 ↓
                    ▼ 最小伤害模式 (贡献3) → MINIMUM_HARM → 输出
```

## 6. 与现有方法的对比

| 维度 | RSS[4] | CBF[5] | 本方法 |
|------|--------|--------|--------|
| 风险粒度 | 单层（安全距离） | 单层（屏障函数） | **三层（空间+交互+事件）** |
| 多体交互 | 无 | 无/简单 | 成对冲突图 |
| 未来可行性 | 无 | 无 | **前瞻仿真检查** |
| 极端场景 | 失效 | 失效 | **最小伤害兜底** |
| 跨算法复用 | 需适配 | 需适配 | **独立接口** |

## 7. 实验设计

### 7.1 数据集

| 数据集 | 类型 | 规模 | 用途 |

### 7.2 对比方法

| 方法 | 类型 |
|------|------|
| NoConstraint | 无约束基线 |
| RSS-Only [4] | 经典安全距离约束 |
| CBF-Based [5] | 控制屏障函数约束 |
| Ours | 完整三层约束 |

### 7.3 消融变体

| 变体 | 去除内容 | 验证目标 |
|------|----------|----------|
| Ours w/o RiskEvents | 去除事件层 | 事件枚举的贡献 |
| Ours w/o RiskGraph | 去除交互层 | 交互分析的贡献 |
| Ours (RiskMap only) | 仅保留空间层 | 三层必要性 |
| Ours w/o FeasCheck | 去除可行性检查 | 长远可行性的贡献 |
| Ours w/o MinHarm | 去除最小伤害 | 极端场景处理的贡献 |

### 7.4 评估指标

| 指标 | 含义 |
|------|------|
| Feasible Area (m²) | 安全动作空间面积（越大=效率越高） |
| Constraint Ratio | 约束收紧比例（1 - 安全面积/无约束面积） |
| Min TTC (s) | 最小碰撞时间（安全余量） |
| Min-Harm Rate | 最小伤害模式触发比例 |
| Compute Time (ms) | 单帧计算延迟 |
