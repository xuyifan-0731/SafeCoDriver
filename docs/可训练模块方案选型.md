# 可训练模块方案选型（详细版）

## 总体思路

在规则驱动的安全约束骨架之上，引入可训练模块提升效果和创新性。按两个维度设计：

1. **底层感知+风险**：替代手工公式（方案 A/C/D）
2. **上层监管+辅助**：大模型长周期预警（方案 E）

---

## 方案 A：学习型风险评估网络

**已实现**（`coop_safety/learned/`），13K 参数。

**当前问题**：用规则蒸馏训练，93.5% 标签为 0（极度不平衡），val loss 几乎不降。

**改进方向**：
- 用 CARLA 仿真的碰撞数据训练（但 CARLA 收集到 0 碰撞——aggressive traffic 不够激进）
- 或用 focal loss 替代 MSE 处理不平衡
- 或用 DAIR-V2X 的 TTC 作为连续风险标签（而非二值碰撞/安全）

**结论**：当前版本效果有限，需要更好的训练数据源。

## 方案 B：学习型约束调节器（替代手工阈值）

### 核心思想

不替代风险评估或约束流程本身，而是学习一组**场景自适应的超参数**。不同场景（高速公路 vs 城市路口 vs 学校区域）应使用不同的 TTC 阈值、排除 buffer、风险等级划分。

### 输入输出

```
输入（场景统计特征, 10 维）:
  - ego_speed (m/s)
  - n_agents (count)
  - mean_agent_distance (m)
  - min_agent_distance (m)
  - mean_ttc (s)
  - min_ttc (s)
  - n_pedestrians (count)
  - ego_acceleration (m/s²)
  - traffic_density (agents / 100m²)
  - mean_agent_speed (m/s)

输出（约束参数, 6 维）:
  - ttc_warning (s): TTC 警告阈值 [2.0, 10.0]
  - ttc_critical (s): TTC 临界阈值 [1.0, 5.0]
  - buffer_scale: agent 排除 buffer 乘数 [0.5, 3.0]
  - high_risk_threshold: RiskMap HIGH 阈值 [0.1, 0.8]
  - medium_risk_threshold: RiskMap MEDIUM 阈值 [0.05, 0.5]
  - approach_weight: 接近速度权重 [0.3, 2.0]
```

### 网络架构

```python
class ConstraintAdjuster(nn.Module):
    def __init__(self):
        super().__init__()
        self.mlp = nn.Sequential(
            nn.Linear(10, 32),
            nn.ReLU(),
            nn.Linear(32, 16),
            nn.ReLU(),
            nn.Linear(16, 6),
            nn.Sigmoid(),  # 输出 [0, 1]，后续缩放到各参数范围
        )
    # 参数量: ~1000
```

### 可行的训练方法

**方法 1: 离线优化（推荐，最简单）**

在 DAIR-V2X 6617 帧上，对每帧暴力搜索最优参数：
```
for each frame:
    for each parameter combination (grid search 或 random search):
        run constraint pipeline with these parameters
        compute score = α × constraint_ratio + β × safety_margin - γ × mode_penalty
    record: scene_features → best_parameters
train MLP to predict: scene_features → best_parameters
```

这是一种**模仿学习**：模仿"穷举出的最优参数"。

代价：6617 帧 × ~100 参数组合 × 0.5s/次 = ~92 小时（太慢）。
加速：随机采样 500 帧 × 30 组合 = ~2 小时。

**方法 2: 在线强化学习**

定义 reward:
```
reward = w1 * constraint_ratio       # 鼓励更多约束
       + w2 * min_ttc_improvement   # 鼓励提高安全余量
       - w3 * over_constraint       # 惩罚可行域过小（<50m²）
       - w4 * mode_penalty          # 惩罚进入 min-harm 模式
```

用 PPO 或 simple policy gradient 优化 MLP 参数。
环境 = 每帧的约束计算（不需要仿真器，离线就能跑）。

代价：约 500 帧 × 1000 episodes × 0.3s = ~42 小时（也慢）。

**方法 3: 分析解（最快，不需要训练）**

不用神经网络，而是用**规则映射**：
```python
if min_agent_distance < 10m:
    ttc_warning = 6.0  # 更保守
elif n_pedestrians > 2:
    ttc_warning = 5.0
else:
    ttc_warning = 4.0
```

这不算"可训练模块"，但可以作为 ablation 的参照。

### 结论

**推荐方法 1**（离线优化+模仿学习），但需要解决计算量问题。可以通过只采样 200 帧 + 10 组参数来加速到 ~20 分钟。

## 方案 C：学习型轨迹风险预测器

（替代 RiskEvents 的手工碰撞类型分类和严重程度估计）

### 输入
- 每个 agent 的当前状态 + 过去 5 帧历史（如果有连续帧）
- Ego 状态 + 历史

### 网络
```
Agent History Encoder: GRU(input=5, hidden=32) per agent → 32-dim temporal embedding
Interaction: attention over all agents → interaction-aware embeddings
Collision Predictor: MLP(64→32→3) → (collision_prob, ttc, severity)
```

### 训练数据
- 正样本：CARLA 中真实碰撞帧 + 前 10 帧的轨迹
- 负样本：安全行驶轨迹
- **问题**：CARLA 收集到的碰撞数据不足（需要更极端的场景设定）

### 结论
依赖时序轨迹数据，DAIR-V2X 只有 0.1s 帧间隔的单步对（不是长轨迹），实用性有限。

## 方案 D：对比学习场景编码器

### 作用

训练一个**通用的场景编码器**，将任意交通场景编码为一个固定维度的向量。这个向量可以被所有下游模块使用：
- 风险评估：输入场景向量 → 预测风险
- 参数调节：输入场景向量 → 预测最优参数
- 异常检测：场景向量偏离正常分布 → 预警

### 对比学习的核心思想

不需要标注！只需要定义"相似"和"不相似"：
- **正对**：同一 batch 内 TTC 接近（±1s）的两帧 → 场景编码应相近
- **负对**：TTC 差异大（>5s）的两帧 → 场景编码应远离

### 训练方案

```
Step 1: 对 DAIR-V2X 6617 帧计算真实 min_ttc
Step 2: 组织训练批次
    - 每个 batch 32 帧，按 min_ttc 排序
    - 正对：相邻 TTC 的帧对
    - 负对：TTC 差距大的帧对
Step 3: 训练 Set Encoder
    输入：agent 集合 [N × 8 features]
    网络：MLP(8→64→32) per agent + attention aggregation → 128-dim scene vector
    损失：InfoNCE loss (对比学习标准损失)
Step 4: 冻结 encoder，用于下游任务
```

### 网络架构

```python
class SceneEncoder(nn.Module):
    def __init__(self):
        self.agent_mlp = nn.Sequential(nn.Linear(8, 64), nn.ReLU(), nn.Linear(64, 32))
        self.attention = nn.MultiheadAttention(32, 4)
        self.projector = nn.Linear(32, 128)
    
    def forward(self, agents, mask):
        embeds = self.agent_mlp(agents)  # (B, N, 32)
        # Self-attention
        embeds = embeds.permute(1, 0, 2)  # (N, B, 32)
        attn_out, _ = self.attention(embeds, embeds, embeds, key_padding_mask=~mask)
        attn_out = attn_out.permute(1, 0, 2)  # (B, N, 32)
        # Masked mean pool
        attn_out[~mask] = 0
        scene = attn_out.sum(1) / mask.sum(1, keepdim=True).float()
        return self.projector(scene)  # (B, 128)
    # 参数量: ~50K
```

### 优势
- 无需标注数据（自监督）
- 通用：一个编码器服务多个下游模块
- 理论基础扎实（对比学习在视觉和 NLP 都验证过）

### 劣势
- 额外组件，增加了系统复杂度
- 对比学习的正负对设计影响效果，需要调参

## 方案 E：大模型上层监管（LLM-as-Supervisor）

### 核心思想

大模型（如 GPT-4/Claude）推理延迟 200-500ms，不能逐帧调用。但可以**每 10 秒更新一次**，做**长周期预警和策略校正**。

### 与实时约束系统的分工

| 模块 | 更新频率 | 职责 |
|------|---------|------|
| 规则/学习型约束 | 10-50Hz | 逐帧计算安全约束（实时） |
| **LLM Supervisor** | **0.1Hz (每 10s)** | 场景理解 + 长远预警 + 参数校正 |

### LLM Supervisor 的具体输入输出

```
输入 (每 10 秒): 过去 10 秒的统计摘要
  "In the last 10 seconds:
   - Ego traveled 80m at avg 8 m/s on an urban road
   - Encountered 15 unique agents (12 vehicles, 3 pedestrians)
   - Min TTC was 2.1s at t=5.3s (with vehicle v7, side approach)
   - Entered CONSERVATIVE mode 2 times, MINIMUM_HARM 0 times
   - RiskMap: avg 3 HIGH cells, 8 MEDIUM cells
   - Current scene: approaching intersection, 2 pedestrians near crosswalk"

输出 (结构化 JSON):
  {
    "risk_trend": "increasing",  // 风险趋势
    "predicted_risk_next_10s": "high",  // 未来 10s 预测
    "warnings": [
      "Pedestrians near crosswalk likely to cross in next 5s",
      "Vehicle v7 shows erratic lane changes — increase buffer"
    ],
    "parameter_adjustments": {
      "ttc_warning": 6.0,  // 建议提高 TTC 阈值
      "pedestrian_buffer": 2.0,  // 建议增加行人 buffer
      "approach_weight": 1.5  // 建议增加接近权重
    },
    "confidence": 0.8
  }
```

### 当前大模型在自动驾驶安全中的进展

需要调研的关键工作：
- **DriveGPT4** (2024): 用 GPT-4V 理解驾驶场景并生成决策建议
- **LanguageMPC** (2024, ICRA): 用 LLM 生成 MPC 的约束参数
- **LLM-Assist** (2024, CoRL): LLM 作为高级规划器辅助低级控制
- **Talk2Drive** (2024): 自然语言驾驶指令 → 控制
- **DiLu** (2024): 将驾驶知识编码为 LLM 的提示

**共同趋势**：LLM 不做实时控制，而是做**高级理解、推理和参数建议**——和我们的设计一致。

### 实现可行性

- 可以用本地部署的小 LLM（如 Llama-3-8B）实现 <1s 延迟
- 或用 API 调用（GPT-4/Claude），每 10s 一次成本极低
- 降级策略：LLM 不可用时用默认参数

### 与最小伤害模式的结合

LLM Supervisor 可以在进入 MINIMUM_HARM 之前提供额外信息：
```
LLM: "Vehicle v12 is a school bus that may stop suddenly.
      If forced to choose, steer RIGHT (sidewalk is clear)
      rather than LEFT (oncoming traffic)."
```

这种**高级语义判断**是规则系统和小型网络无法做到的。

---

## 推荐组合

**实时层（10-50Hz）**: 方案 B（学习型参数调节）或方案 A（学习型风险评估）  
**监管层（0.1Hz）**: 方案 E（LLM Supervisor）  

方案 B + E 的组合最有创新性且实际可行：
- B 让约束参数根据场景自适应（快速、轻量）
- E 提供长周期预警和语义理解（慢速但深度）
- 两者互补，且都不需要大量训练数据

## 最小伤害模式详细设计

### 当前实现

```python
# coop_safety/constraint/min_harm.py
class MinimumHarmPlanner:
    def plan(ego, events):
        candidates = [紧急制动] + [制动+左转] + [制动+右转] + ...  # 12个候选动作
        for action in candidates:
            traj = bicycle_model.predict(ego, action, 1.5s)  # 前向仿真
            cost = Σ(severity × probability × proximity × type_penalty)
            # type_penalty: 行人×5, 正面碰撞×3
        return min_cost_action
```

### 借鉴 TCAS 的改进方向

1. **分层紧急程度**：
   - Level 1 (Caution): 减速 30% — 对应轻微风险
   - Level 2 (Warning): 减速 60% + 轻微转向 — 对应中度风险
   - Level 3 (Emergency): 全力制动 + 最优转向 — 对应不可避免碰撞

2. **协调避让**：如果能获取其他车辆的 V2V 通信，可以协调"一个左转一个右转"

3. **后方安全检查**：制动前检查后方车辆距离（已在 CARLA 闭环中实现）
