# 新实验方案：安全约束提升协同自驾性能

## 核心思路转变

**旧思路**（已证明不够）：单独评估安全约束的"约束率"和"TTC"——这些是代理指标，不是最终目标。

**新思路**：将安全约束模块**集成到一个完整的协同自驾系统**中，用**标准自驾评测指标**（碰撞率、路线完成率、舒适度等）证明"加了安全约束的自驾算法比不加的更好"。

## 平台选择

### V2Xverse (TPAMI 2025)

**论文**: "Towards Collaborative Autonomous Driving: Simulation Platform and End-to-End System"  
**GitHub**: https://github.com/CollaborativePerception/V2Xverse (★180)

**选择原因**:
1. 协同感知 + 端到端自驾的完整 pipeline
2. 支持 CARLA 闭环评估
3. 已集成多个协同感知方法（V2X-ViT、V2VNet、F-Cooper）
4. 已集成多个端到端自驾方法（Transfuser、InterFuser、TCP）
5. 有标准化的评测指标和基线结果
6. 2025 年发表在 TPAMI，足够新

### 备选数据集：DeepAccident

**论文**: Wang et al., "DeepAccident: A Motion and Accident Prediction Benchmark for V2X Autonomous Driving", 2023  
**GitHub**: https://github.com/tianqi-wang1996/DeepAccident (★96)

**特点**:
- 专门针对**碰撞场景**的 V2X 数据集
- 包含 accident 和 normal 两类场景
- V2XFormer 模型支持多车协同感知+预测
- 评测指标包括 motion prediction + accident prediction

**用法**: 用 DeepAccident 的碰撞场景数据**训练**我们的风险评估网络（解决之前"93.5%标签为0"的问题），用 V2Xverse 做**评测**。

## 实验设计

### 实验 1: 安全约束提升协同自驾闭环性能

```
基础算法: V2Xverse 的 CoDriving 系统 (V2X-ViT + TCP/Transfuser)
          ↓
加入安全约束: CoDriving + Ours safety constraint
          ↓
评测: CARLA 闭环驾驶指标
```

**对比设定**:
| 配置 | 说明 |
|------|------|
| CoDriving (base) | V2Xverse 原始系统 |
| CoDriving + RSS | 加 RSS 安全约束 |
| CoDriving + CBF | 加 CBF 安全约束 |
| CoDriving + RPF | 加 RPF 安全约束 |
| **CoDriving + Ours** | **加我们的三层安全约束** |

**评测指标** (V2Xverse 已有):
- Route Completion (RC): 路线完成率
- Driving Score (DS): 综合驾驶得分
- Collision Rate: 碰撞率
- Red Light Infraction: 闯红灯次数
- Comfort: 舒适度（加速度平滑性）

### 实验 2: 碰撞场景训练

用 DeepAccident 的碰撞数据训练风险评估网络：
- accident 场景 → 高风险标签
- normal 场景 → 低风险标签
- 比规则蒸馏的标签分布更平衡

### 实验 3: 消融和对比

在相同的 CoDriving 基础上：
- 消融三层风险的各层贡献
- 对比有/无协同感知下安全约束的效果
- 对比规则版 vs 学习版风险评估

## 需要的工作

### Step 1: 搭建 V2Xverse 环境
- 需要 CARLA 0.9.10.1（我们有 0.9.15，可能需要降级或兼容）
- 下载 V2Xverse 代码和预训练模型
- 验证基线能跑通

### Step 2: 集成安全约束
- 在 V2Xverse 的 planning 模块之后、control 之前插入安全约束
- 安全约束接收 V2X 感知结果，输出约束后的动作空间
- planning 模块的输出如果超出安全空间则被投影回来

### Step 3: 下载 DeepAccident 数据
- 用碰撞场景训练风险网络
- 约 50GB 数据

### Step 4: 运行评测
- V2Xverse 闭环评测 pipeline

## 风险

1. **CARLA 版本兼容**: V2Xverse 需要 0.9.10.1，我们有 0.9.15。可能需要重新下载旧版
2. **V2Xverse 环境复杂**: 依赖 mmdet3d、open3d、spconv 等，可能有冲突
3. **计算资源**: 闭环评测需要同时跑 CARLA + 感知 + 规划，GPU 占用较高
4. **DeepAccident 数据量大**: ~50GB，下载耗时
