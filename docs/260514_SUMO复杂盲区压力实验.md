# 260514 SUMO 复杂盲区压力实验记录

## 实验设计更新

- 基础集：沿用已修正的 `BS1..BS5 x blind/nonblind` 共 10 个 SUMO 场景。
- 压力集：在 `BS1..BS4 x blind/nonblind` 中增加 `attacker_rear`，用于验证避让主冲突后是否引发后向追尾/二次碰撞。
- `CollRate` 只统计 SUMO `traci.simulation.getCollisions()` 中 ego 参与的真实碰撞。
- `OverlapRate` 仅保留为轨迹矩形重叠诊断，不计入碰撞率。
- `2ndC` 统计同一次 run 中 ego 的第二个及之后不同碰撞 pair。
- `severity` 用碰撞 pair 的相对速度均值表示。
- 新增 `collision_pairs` 到 `per_run.csv`，用于区分 `attacker+ego` 主冲突和 `attacker_rear+ego` 追尾。

## 新增算法变体

- `Hybrid+AND+TTC`：保留 AND 融合降低误报，但当前/侧向 TTC 小于 3s 时强制触发，修复原 `Hybrid+AND` 在 BS3 的漏检。
- `Hybrid+AND+TTC+RearAware`：区分前/侧向 TTC 与后向 TTC，后向追尾风险主导时抑制强刹。
- `Hybrid+AND+TTC+MinHarm`：在近距离后车存在时进一步限制急刹，除非前/侧向 TTC 已低于 1s，用于降低不可避免碰撞下的严重度。
- `Hybrid+AND+TTC+RearEscape`：在后车近距离追尾风险主导时优先尝试相邻空车道逃逸；若邻道 25m 内有车，则不换道，避免把追尾风险转化为二次碰撞。
- SUMO 闭环控制增加对几何修正路点的换道意图映射，使 Hybrid 的 waypoint correction 不只体现在 WPC 指标中。

## 对比方法完整性检查

本轮 v7 key 实验包含 12 个方法：

- 无约束：`NoCon-egoonly`, `NoCon-coop`
- 规则/几何/学习基线：`RSS-coop`, `UniE2EV2X-coop`, `MAP-coop`, `RiskMM-coop`
- 本文方法及改进：`Hybrid-thr0.30`, `Hybrid+AND-thr0.30`, `Hybrid+AND+TTC-thr0.30`, `Hybrid+AND+TTC+RearAware-thr0.30`, `Hybrid+AND+TTC+MinHarm-thr0.30`, `Hybrid+AND+TTC+RearEscape-thr0.30`

完整配置函数中仍保留 `0.25/0.30/0.40` 三个阈值；v7 key 表格使用 `thr=0.30` 作为代表配置，以便快速比较压力场景中的闭环碰撞行为。

压力集观察：

- BS3/BS4 中，RSS/RiskMM/Hybrid 的碰撞 pair 多为 `attacker_rear+ego`，说明强刹避开主攻击车后，会被新增后车高速追尾。
- `Hybrid+AND+TTC` 修复了基础集漏检，但在压力集会触发更强刹车，导致严重度升高。
- `MinHarm` 没有降低压力集 CollRate，但把 Ours 在压力集的平均严重度从 13.50 降到 10.25，并保持 0 二次碰撞。
- `RearEscape` 进一步把压力集 CollRate 从 62% 降到 25%，并保持 0 二次碰撞；剩余失败集中在 BS4，pair 为 `attacker_rear+ego`，说明相邻车道被 `blk_right` 占用后只能回退到纵向 minimum-harm 控制。
- MAP 已纳入表格。压力集结果为 `CollRate=100%`, `2ndC=0`, `Sev=10.05`, `Det(s)=25%`, `FA(f)=0.0%`。其主要失败原因是 bbox 后处理只改路点/低频刹车，无法对强制横穿和后车追尾形成及时闭环避让。
- 当前复杂压力集已经能让所有关键方法产生一次碰撞机会；二次碰撞仍主要只在 NoCon 的 BS4 出现。若论文需要“所有方法都有二次碰撞样本”，下一步应增加第二个受控后向/侧向攻击车，而不是只靠 SUMO car-following 自然传播。

## 推荐结论

- 普通 SUMO 10 场景最优：`Hybrid+AND+TTC-thr0.30`，0% CollRate、0 二次碰撞、1.4% FA(f)。
- 同时考虑复杂压力场景：若优先级为 CollRate > 二次碰撞 > 严重度，则 `Hybrid+AND+TTC+RearEscape-thr0.30` 最优；若更强调不可避免碰撞的平均严重度，则 `Hybrid+AND+TTC+MinHarm-thr0.30` 更保守。
- 算法改进方向：碰撞决策不能只用全局最小 TTC 强刹，需要把 TTC 分解为前/侧向和后向，并引入 rear-gap guard 与 minimum-harm 策略，避免把主冲突风险转化为高速追尾风险。
