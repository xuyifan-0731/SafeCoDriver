# 方法实验说明 (2026-04-28)

---

## 一、算法改进内容

### 1.1 改进动机

v1 (CollisionPredictionNetwork) 存在的问题：
1. **路点碰撞率高 (2.9%)**：与 NoConstraint 相同——没有修改危险路点的能力
2. **提前预警帧不够 (27.8)**：低于 UniE2EV2X (57.2) 和 RiskMM (60.0)
3. **网络设计缺乏创新性**：简单 MLP + 对称 self-attention，没有利用盲区信息

### 1.2 数据分析发现

| 发现 | 数据 | 改进方向 |
|------|------|---------|
| accident 场景有大量 invisible agents | accident: 1329, normal: 0 | **可见性是关键区分特征** |
| 正常场景 69.6% 帧有 agent <10m | 密集≠危险 | 不能只靠距离判断 |
| 距碰撞 50+ 帧时也有 30-40% agents <5m | 交互模式比距离更重要 | 需要 approach speed 特征 |

### 1.3 CollisionPredictionNetV2 架构 (66,291 参数)

```
输入层:
  agents: (B, N, 12) — 相对特征
    [rel_x, rel_y, rel_vx, rel_vy, heading, length, width, speed,
     visible(0/1), type_id, approach_speed, dist_to_ego]
  ego: (B, 6) — [speed, yaw_rate, ax, ay, heading, 0]
  waypoints: (B, 10, 2) — 待评估的候选路点

RelativeAgentEncoder (10,512 params):
  ├── spatial_mlp: Linear(8→64) + GELU + Linear(64→64) — 空间特征
  ├── visibility_embed: Embedding(2, 16) — 可见/不可见分开编码
  ├── type_embed: Embedding(6, 8) — 车型编码
  └── fuse: Linear(64+16+8=88→64) + GELU — 融合

EgoCentricAttention (25,536 params):  ← 创新点
  ├── ego_proj: Linear(6→64) — ego → query
  ├── agent_k/v: Linear(64→64) — agents → key/value
  ├── attn: MultiheadAttention(64, 4heads) — ego 查询 agents
  └── agent_self_attn: MultiheadAttention — agent 间多体交互
  输出: scene_vector(64), agent_interacted(N,64), attention_weights(N)

SceneProjection (8,320 params):
  Linear(64→128) + GELU

CollisionHead (8,321 params):
  Linear(128→64) + GELU + Dropout(0.1) + Linear(64→1) → sigmoid
  输出: P(collision within 30 frames)

TTCHead (8,321 params):
  Linear(128→64) + GELU + Linear(64→1) → relu
  输出: estimated TTC (seconds)

WaypointRiskScorer (5,281 params):  ← 创新点
  输入: [scene_vec(128); waypoint_xy(2)] = 130
  Linear(130→32) + GELU + Linear(32→32) + GELU + Linear(32→1) → sigmoid
  输出: per-waypoint risk score [0,1]
```

### 1.4 设计创新点

| 创新 | 描述 | 对应 v1 问题 |
|------|------|-------------|
| **可见性感知编码** | invisible agents 用独立 embedding，作为盲区危险信号 | v1 不区分 visible/invisible |
| **Ego-Centric Attention** | ego 作为 query 查询 agents（不对称），建模"各 agent 对 ego 的威胁" | v1 用对称 self-attention |
| **Approach Speed 特征** | 相对速度在连线方向的投影，正值=接近 | v1 只有绝对速度 |
| **Per-Waypoint Risk Scoring** | 对每个候选路点输出风险分数，直接指导路点修正 | v1 只输出全局碰撞概率 |
| **Multi-task Learning** | 碰撞预测 + TTC + 路点风险联合训练 | v1 只有碰撞+TTC |

## 二、训练配置

### 2.1 数据

| 项 | 值 |
|----|-----|
| 数据集 | DeepAccident val (104 scenarios, 8193 frames) |
| 特征维度 | 12 per agent (含 visibility, approach_speed, dist_to_ego) |
| 最大 agents | 30 per frame |
| 路点数 | 10 (0.5s 间隔, 共 5s) |
| 正样本 | accident 场景距碰撞 ≤30 帧 |
| 路点风险标签 | 路点距任何 agent 预测位置 <3m → risk=1 |
| 划分 | 80% train, 20% val (shuffle 后) |

### 2.2 训练参数

| 项 | 值 |
|----|-----|
| Optimizer | AdamW, lr=3e-4, weight_decay=1e-4 |
| Scheduler | CosineAnnealingLR, T_max=100 |
| Epochs | 100 |
| Batch size | 64 |
| 正样本过采样 | 最多 4× 重复 |
| 梯度裁剪 | max_norm=1.0 |

### 2.3 损失函数

```
L = L_collision + 0.1 × L_ttc + 0.2 × L_waypoint_risk

L_collision = Focal BCE (γ=2.0, α=0.75)
  - γ=2.0: 聚焦难分样本（接近碰撞但不确定的帧）
  - α=0.75: 正样本权重更高（12.7% 正样本不平衡）

L_ttc = SmoothL1Loss(predicted_ttc, gt_ttc)
  - 回归任务，预测距碰撞的时间

L_waypoint_risk = BCE(predicted_wp_risk, gt_wp_risk)
  - 对每个路点独立预测风险
  - gt_wp_risk = 1 if 路点距 agent <3m at predicted time else 0
```

### 2.4 训练结果

| 版本 | AUC | Precision | Recall | F1 | 参数量 |
|------|-----|-----------|--------|----|----|
| v1 (CollisionPredNet) | **0.985** | 0.885 | 0.954 | 0.918 | 46K |
| **v2 (CollisionPredNetV2)** | **0.978** | 0.791 | 0.926 | 0.854 | 66K |

v2 的 AUC 略低于 v1 (0.978 vs 0.985)，因为：
1. 多了 waypoint_risk 任务分散了优化目标
2. 更复杂的特征编码需要更多数据
3. EgoCentricAttention 中的动态模块增加了训练难度

但 v2 新增了 v1 没有的能力：**per-waypoint risk scoring**（可直接修正危险路点）。

### 1.5 Neural-Geometric Hybrid 安全约束 (Ours-Hybrid)

**动机**: v1/v2 虽然碰撞检测率高 (100%)，但路点碰撞率 (WPColl%) 与 NoConstraint 相同 (2.9%)——因为只输出约束模式，不修改路点。对比方法 MAP (0.4%) 和 UniE2EV2X (0.7%) 通过几何碰撞检测+路点推移实现了低碰撞率。

**核心思想**: 结合 V2 网络的学习型风险评估与增强型几何碰撞检测，取各方法之长并引入新创新。

**三个创新点**:

| 创新 | 描述 | vs 对比方法 |
|------|------|-----------|
| **盲区感知安全缓冲** | 不可见 agent (仅通过 V2X 检测) 使用更大安全边距 (4.0m vs 2.5m)，因为位置不确定性更高 | UniE2EV2X/MAP 对所有 agent 使用相同阈值 |
| **接近速度自适应碰撞区** | 安全边距随 agent 接近速度增大: margin × (1 + 0.3 × approach_speed / v_max) | UniE2EV2X/MAP 使用固定碰撞区 |
| **多智能体排斥场路点修正** | 对每个路点累加所有威胁 agent 的排斥力，防止推向另一个 agent | UniE2EV2X/MAP 只处理最近的一个碰撞 |

**流程**:
```
输入: base_waypoints (10, 2) + perception (agents + ego)

Stage 1: 神经风险评估 (V2 网络)
  → collision_prob, per-waypoint risk scores
  → 用于: 检测触发、提前预警

Stage 2: 可见性感知几何碰撞检测
  → 对每个路点:
    a. 预测所有 agent 在 t 时刻的位置
    b. 计算安全边距: visible → 2.5m, invisible → 4.0m
    c. 接近速度缩放: margin × (1 + 0.3 × approach / v_max)
    d. 检测所有 < margin 的 agent

Stage 3: 多智能体排斥场路点修正
  → 对每个有碰撞的路点:
    a. 累加所有威胁 agent 的排斥向量
       F_i = (wp - agent_i) / |wp - agent_i|² × weight_i
       weight_i 考虑可见性和网络风险分数
    b. 归一化方向，按碰撞深度确定推移量

Stage 4: 轨迹平滑 (weight=0.3)
Stage 5: 速度约束 (max 20m/s)
```

**代码文件**: `coop_safety/learned/hybrid_safety.py` (HybridSafetyConstraint 类)

### 3.3 评价指标

#### 碰撞检测指标

| 指标 | 计算方式 | 理想值 |
|------|---------|--------|
| **Detection Rate** | (约束在碰撞前触发的 accident scenarios) / (总 accident scenarios) | 100% |
| **Early Warning Frames** | 首次触发预警的帧 到 scenario 结束帧 的帧数差 | 越大越好 |
| **False Alarm Rate** | (约束在 normal scenarios 中误触发的 scenarios) / (总 normal scenarios) | 0% |
| **Modification Rate** | (被约束修改的帧数) / (总帧数) | 越低越好（精准介入） |

- Detection Rate: 对每个 accident scenario，检查约束是否在 scenario 期间触发过 CONSERVATIVE 或 MINIMUM_HARM 模式。触发=检测成功。
- Early Warning: 首次触发的帧索引 fi_first，scenario 总帧数 n_frames，early = n_frames - fi_first。
- False Alarm: 对每个 normal scenario，检查约束是否触发过。触发=误报。

#### 路点碰撞指标

| 指标 | 计算方式 | 理想值 |
|------|---------|--------|
| **Waypoint Collision Rate** | (与 agent 距离<2m 的路点数) / (总路点数) | 0% |

- 对每帧的 10 个路点，检查每个路点在对应时间步的位置是否与任何 agent 的预测位置距离 <2m。
- 计算: `wp_coll = Σ(1 if min_dist(wp_t, agents_at_t) < 2m) / (n_frames × 10)`

#### 约束质量指标（SafeActionSpace 接口方法）

| 指标 | 计算方式 |
|------|---------|
| **Constraint Ratio** | 1 - feasible_area / unconstrained_area |
| **Min TTC** | 最小碰撞时间 |
| **Mode Distribution** | NORMAL/CONSERVATIVE/MINIMUM_HARM 各占比例 |

### 3.4 实验流程

```
对每个 scenario (104个):
  对每帧 (50-80帧):
    1. 加载 GT 感知 (DeepAccidentLoader)
    2. 生成 baseline waypoints (模拟 CoDriving: constant-velocity)
    3. 对每个方法:
       a. 传入感知 → 安全约束 → 输出修正后 waypoints
       b. 检查约束模式 (NORMAL/CONSERVATIVE/MINIMUM_HARM)
       c. 检查修正后 waypoints 的碰撞率
    4. 记录所有指标
```
