# 方法改进方案 (2026-04-24)

## 当前问题总结

1. **V2Xverse 过度修正** (ADE 7.3) — 全局投影策略错误
2. **DeepAccident 误报率 100%** — 阈值过敏感
3. **学习标签不平衡** (93.5% 为 0) — 用规则蒸馏天花板低
4. **缺乏碰撞预测能力** — 当前方法是"约束"而非"预测"

## 改进方案

### 1. 数据基础：全部切换到 DeepAccident

- **训练集**: type1_subtype1 + type1_subtype2 (52 accident + 52 normal = 104 scenarios, ~5000 帧)
- **GT 感知**: 直接用 label 中的 3D 框 (位置/速度/尺寸/类型)
- **盲区处理**: label 中 `visible=True/False` 直接标记可见性；`other_vehicle/label/` 提供协同感知补全
- **碰撞监督**: 第 12 字段 (collision countdown) > 0 → 高风险帧；= 0 → 安全帧
- **标签平衡**: accident 场景中约 40% 帧有碰撞标记 → 标签分布远好于 DAIR-V2X

### 2. 网络改进：碰撞预测网络（替代风险评估网络）

**动机**: 当前 RiskAssessmentNetwork 学的是"空间风险分数"（连续值 0-1，大部分为 0）。改为直接预测"未来 T 帧是否碰撞"（二分类，标签来自 GT collision countdown）。

**新架构: CollisionPredictionNetwork**

```
输入:
  agents: (B, N, 10)  [x, y, vx, vy, heading, length, width, type, collision_countdown, visible]
  ego: (B, 6)  [speed, yaw_rate, x, y, heading, 0]

TemporalEncoder (如果有连续帧):
  将 T 帧历史编码为时序特征
  GRU(input=10, hidden=64) per agent → 64-dim temporal embedding
  或简化: 只用当前帧 (无历史)

AgentInteraction:
  Multi-head Self-Attention (4 heads, dim=64) over agents
  捕捉 agent 间的交互关系 (A 减速 → B 不会撞 C)

SceneEncoder:
  Attention-pooling over agents → 128-dim scene vector

输出 heads:
  1. CollisionHead: MLP(128→64→32→1) → sigmoid → P(collision in next T frames)
  2. RiskMapHead: MLP(128+2→64→32→1) → sigmoid → R(x,y) per query point
  3. TTCHead: MLP(128→64→1) → relu → estimated min TTC
```

**参数量**: ~50K (仍然极轻量)

**训练**:
- 正样本: collision_countdown > 0 的帧 → label=1
- 负样本: 正常场景 + accident 场景中 countdown=0 的帧 → label=0
- Loss: Focal Loss (处理不平衡) + BCE(collision) + MSE(ttc)
- 数据: DeepAccident 104 scenarios × ~50 frames = ~5000 samples

### 3. 约束逻辑改进

**问题**: 当前约束的核心问题是"一旦检测到风险就大面积排除可行域"。

**改进: 分级响应 + 碰撞预测引导**

```
Level 0 (P(coll) < 0.3): NORMAL — 不干预
Level 1 (0.3 ≤ P(coll) < 0.6): ADVISORY — 轻微收紧可行域 (排除碰撞方向 10%)
Level 2 (0.6 ≤ P(coll) < 0.8): CONSERVATIVE — 中度收紧 (排除碰撞方向 40%)
Level 3 (P(coll) ≥ 0.8): MINIMUM_HARM — 强力约束 (最小伤害选择)
```

**关键改变**: 约束强度与碰撞概率**连续关联**，而非硬阈值触发。

### 4. V2Xverse 集成改进

**问题**: 全局投影策略导致 ADE 恶化。

**改进: 碰撞检测 + 局部偏移**

```python
for each waypoint (t, x, y):
    min_dist = min distance to any detected object at time t
    if min_dist < safety_margin (2m):
        # Only push this waypoint away from collision
        push_direction = away from nearest object
        push_amount = safety_margin - min_dist
        waypoint += push_direction * push_amount
    # else: don't modify this waypoint
```

### 5. 关于分层强化学习

**分层 RL 适用性分析**:

分层 RL (Hierarchical RL, HRL) 的核心是将复杂任务分解为高层策略（目标选择）和低层策略（动作执行）。

在我们的场景中：
- **高层策略**: 决定"当前场景的约束级别"(NORMAL/ADVISORY/CONSERVATIVE/MINIMUM_HARM)
- **低层策略**: 在给定约束级别下，选择具体的可行域参数 (buffer_size, ttc_threshold 等)

**可行但不推荐作为首选**，原因：
1. 需要闭环仿真环境定义 reward（当前 CARLA 0.9.10 无法运行）
2. 训练样本效率低（需要大量 episode）
3. 我们的任务更适合监督学习（有明确的 GT 碰撞标签）

**更推荐的替代方案**:

| 方案 | 适用性 | 原因 |
|------|--------|------|
| **监督学习碰撞预测** | ★★★ | DeepAccident 有完美的碰撞标签，直接训练 |
| **对比学习场景编码** | ★★☆ | 无需标签，区分危险/安全场景 |
| 分层 RL | ★☆☆ | 需要仿真环境 + reward 设计，当前条件不足 |
| 模仿学习 | ★★☆ | 模仿规则系统的最优参数，但天花板是规则系统 |

**结论**: 优先实现监督学习碰撞预测 (方案 2)。DeepAccident 的 GT 碰撞标签是天然的监督信号，直接学习"什么场景会碰撞"比学习"如何调参数"更直接有效。
