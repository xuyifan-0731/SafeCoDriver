# 对比方法详细介绍

> 本文档详细描述三个新增对比方法的安全约束机制，包括论文出处、算法原理、是否存在安全约束、输入输出、以及复现方案。

---

## 1. UniE2EV2X — 基于规则几何碰撞判定的事故预警

### 论文信息

| 项 | 值 |
|----|-----|
| 标题 | Unified End-to-End V2X Cooperative Autonomous Driving |
| 作者 | Zhiwei Li, Bozhen Zhang, Lei Yang, Tianyu Shen, Nuo Xu, Ruosen Hao, Weiting Li, Tao Yan, Huaping Liu |
| 年份 | 2024 |
| arXiv | https://arxiv.org/abs/2405.03971 |
| 数据集 | DeepAccident |
| 开源代码 | 未公开 |

### 安全约束机制：几何碰撞后处理

**是否存在安全约束**：是，但不是传统的"约束可行域"，而是**基于运动预测的几何碰撞后判定**。

**算法**：
1. 端到端模型输出所有 agent（含 ego）的未来 T 帧运动轨迹
2. 对每个时间步，将每个 agent 的预测位置近似为多边形（bounding box）
3. 检查每对 agent 之间的最小距离是否低于**安全阈值**
4. 若低于阈值 → 标记为碰撞事件（输出碰撞的 agent ID、位置、时间戳）

**关键公式**：
```
对每个时间步 t：
  对每对 agent (i, j)：
    polygon_i = approx_polygon(position_i(t), size_i)
    polygon_j = approx_polygon(position_j(t), size_j)
    min_dist = min_distance(polygon_i, polygon_j)
    if min_dist < safety_threshold:
      collision_detected(agent_i, agent_j, t)
```

**输入**：预测的运动轨迹（position + size per agent per timestep）
**输出**：碰撞事件列表（agent IDs, positions, timestamps）

**特点**：
- 不修改规划轨迹，只做碰撞**检测和预警**
- 依赖上游运动预测的准确性
- 安全阈值是固定的（论文未给出具体数值，需从实验中估计）

### 复现方案

从我们方法的角度，可以复现为：
1. 用 GT 或预测的运动轨迹
2. 对每对 agent 计算每个时间步的几何距离
3. 距离 < threshold → 碰撞预警
4. 如果检测到碰撞 → 将涉及碰撞的区域从可行域中排除（作为约束）

## 2. MAP — 基于地图可行域先验的碰撞感知轨迹选择

### 论文信息

| 项 | 值 |
|----|-----|
| 标题 | MAP: End-to-End Autonomous Driving with Map-Assisted Planning |
| 作者 | Huilin Yin, Yiming Kan, Daniel Watzenig |
| 年份 | 2025 (CVPR Workshop MEIS) |
| arXiv | https://arxiv.org/abs/2509.13926 |
| 数据集 | DAIR-V2X-seq-SPD |
| 开源代码 | 论文声称"available at this URL"但链接未找到 |

### 安全约束机制：Collision Loss + 地图可行域约束

**是否存在安全约束**：是，通过**碰撞损失函数 (Collision Loss)** 和**地图先验**实现。

**算法**：

**Collision Loss（训练时约束）**：
```
对每个未来时间步 i：
  构建 ego 的预测 bounding box B_hat(i) = box(x_hat(i), y_hat(i), heading_gt(i))
  对每个 GT 障碍物 bbox B_j(i)：
    如果 B_hat(i) 和 B_j(i) 有交集 → 计算碰撞惩罚
  collision_loss(i) = Σ_j penalty(B_hat(i), B_j(i))

总碰撞损失 = Σ_i m(i) × collision_loss(i)
```
其中 m(i) 是 binary mask 指示该时间步是否有有效预测。

**Map-Assisted Feasible Region（推理时约束）**：
- 利用在线地图分割（语义地图）确定可行驶区域
- Plan-enhancing Online Mapping 模块将地图特征注入规划
- Off-road rate 作为评测指标（轨迹是否偏出道路）

**总损失**：
```
L_adaptive = α × L_L2 + β × L_collision + γ × L_offroad
```

**输入**：预测轨迹 + GT 障碍物 bounding boxes + 语义地图
**输出**：受约束的轨迹（通过损失函数隐式约束，非显式可行域）

**特点**：
- 安全约束在**训练时**通过 loss 实现，不是**推理时**的显式约束
- 地图信息提供道路可行域先验
- 碰撞检测基于 bounding box 交集

### 复现方案

将 MAP 的安全约束抽取为推理时的后处理模块：
1. 对预测的轨迹路点，构建 ego bounding box
2. 检查与 GT/预测障碍物的 bounding box 是否碰撞
3. 碰撞时，将路点推离碰撞物体（最小位移使 bbox 不再交叉）
4. 如果有地图信息，将路点约束在道路边界内

## 3. RiskMM — 基于学习的 MPC 约束与风险图中间件

### 论文信息

| 项 | 值 |
|----|-----|
| 标题 | Risk Map As Middleware: Towards Interpretable Cooperative End-to-end Autonomous Driving for Risk-Aware Planning |
| 作者 | Mingyue Lei, Zewei Zhou, Hongchen Li, Jiaqi Ma, Jia Hu |
| 年份 | 2025 |
| arXiv | https://arxiv.org/abs/2508.07686 |
| 数据集 | V2XPnP-Seq |
| 开源代码 | 论文声称将公开，目前未找到 |

### 安全约束机制：Risk Map → Learning-based MPC

**是否存在安全约束**：是，这是三篇中安全约束最完整的——通过**学习型风险图**引导**基于 MPC 的规划器**。

**算法**：

**Step 1: Risk Map 生成**
- 输入：协同感知的 BEV 特征（含 ego 和其他 agent 的占据预测）
- Multi-head Cross-Attention 建模 ego 与环境的交互
- 输出：注意力权重 W_r ∈ R^{1×N×(H×W)}，即风险图
- 风险图是可学习的，没有 GT 标签，端到端训练

**Step 2: Risk Map → Cost Map**
- 通过感知机层将风险特征 F_r 解码为 cost map Z ∈ R^{N×T×M}
- Z 作为 MPC 的加权参数

**Step 3: Learning-based MPC Planner**
- 系统状态：X_k = [s_k, v_k, a_k, θ_k]（位置、速度、加速度、航向）
- 控制输入：U_k = [j_k, κ_k]（加加速度、曲率变化率）
- 目标函数：二次型 J = Σ_k (X_k^T Q_k X_k + U_k^T R_k U_k)
  - Q_k, R_k 由 cost map Z 参数化（可学习权重）
- **显式速度约束**：|v_k| < v_max (80 mph)
- 用 Lagrange 方法求解非线性优化

**输入**：协同感知 BEV 特征
**输出**：满足物理约束和风险感知的规划轨迹

**特点**：
- 风险图从数据中学习（不需要 GT 风险标签）
- MPC 提供物理约束（车辆动力学）和速度限制
- 可解释性：MPC 参数对应明确的物理含义
- 协同感知扩展了感知范围，减少盲区

### 复现方案

将 RiskMM 的安全约束抽取为独立模块：
1. **Risk Map**：用注意力机制建模 agent 间交互 → 输出每个位置的风险值
   - 简化实现：用我们的 RiskMap + 交互注意力替代 BEV 特征
2. **MPC 约束**：
   - 将风险图转为 cost map
   - 用二次型目标函数优化轨迹：J = Σ (x^T Q x + u^T R u)，Q 由风险图参数化
   - 加入速度约束 |v| < v_max 和加速度约束
   - 用 scipy.optimize 或 casadi 求解
3. 与 CoDriving Planner 的集成：
   - CoDriving 输出候选轨迹 → RiskMM 用风险图+MPC 评分/修正 → 输出最终轨迹

## 总结对比

| 方法 | 约束类型 | 约束时机 | 是否显式可行域 | 是否可学习 | 复现难度 |
|------|---------|---------|--------------|-----------|---------|
| **UniE2EV2X** | 几何碰撞检测 | 推理后处理 | 否（只预警） | 否 | 低 |
| **MAP** | Collision Loss + 地图边界 | 训练 loss + 推理后处理 | 部分（地图边界） | 否（loss 是固定公式） | 中 |
| **RiskMM** | 学习型风险图 + MPC | 推理时 MPC 约束 | 是（MPC 可行域） | 是（风险图端到端学习） | 高 |
| **Ours-Rule** | 三层风险 + 分层收紧 | 推理后处理 | 是（多边形可行域） | 否 | — |
| **Ours-Collision** | 碰撞预测网络 + 分级响应 | 推理时概率判断 | 是 | 是（DeepAccident 训练） | — |

---

## 复现优先级

1. **UniE2EV2X 几何碰撞检测**（低难度）：纯几何计算，无需训练
2. **MAP Collision-aware 后处理**（中难度）：bbox 碰撞检测 + 最小位移修正
3. **RiskMM MPC 约束**（高难度）：需实现可微 MPC 求解器

## CoDriving Planner 复现

所有方法（包括 Ours 和三个新增对比）都使用相同的决策模块：**V2Xverse CoDriving Planner**。

| 项 | 值 |
|----|-----|
| 模型 | CoDriving end-to-end planner (V2Xverse, TPAMI 2025) |
| 输入 | 协同感知 BEV 特征 + warped 时序特征 + occupancy map |
| 输出 | future_waypoints: (B, 10, 2) — 10 个未来路点 |
| 权重 | `checkpoints/codriving/planner/codriving_planner.ckpt` (19MB) |
| 评测指标 | ADE (平均位移误差), FDE (终点位移误差), Collision Rate |

**实验流程**：
```
CoDriving Planner → 输出 waypoints
      ↓
安全约束模块 (Ours / UniE2EV2X / MAP / RiskMM / RSS / ...) → 修正 waypoints
      ↓
评测: ADE, FDE, Collision Rate, Off-road Rate
```
