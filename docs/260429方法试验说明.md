# 方法试验说明 (2026-04-29)

---

## 一、研究背景与目标

### 1.1 研究问题

协同感知 (Cooperative Perception, V2X) 为自动驾驶提供了超越单车视野的环境信息。但现有端到端自驾算法（如 CoDriving）缺乏独立的安全约束机制：它们直接从感知结果输出控制/规划，没有系统性的安全保障模块。

**核心目标**：设计一个**独立、可插拔的安全约束模块**，接收协同感知结果，输出安全动作空间约束，可叠加到任意已有自驾规划算法之上。

### 1.2 研究贡献

1. **三层风险评估框架** (RiskMap/RiskGraph/RiskEvents)：从空间、交互、事件三个维度评估碰撞风险
2. **碰撞预测网络** (V1: 46K params)：基于多头注意力的场景级碰撞概率预测
3. **改进碰撞预测网络** (V2: 66K params)：Ego-Centric Attention + 可见性感知编码 + 逐路点风险评分
4. **Neural-Geometric Hybrid 安全约束** (Ours-Hybrid)：V1 检测 + 可见性感知几何修正 + 多智能体排斥场
5. 三个创新技术：盲区感知安全缓冲、接近速度自适应碰撞区、多智能体排斥场路点修正

## 二、算法详细设计

### 2.1 整体架构

```
                     CoDriving Planner
                           │
                     predicted waypoints (10×2)
                           │
              ┌────────────┴────────────┐
              │   Safety Constraint     │
              │   Module (Ours-Hybrid)  │
              │                         │
              │  ┌─── V1 Detector ───┐  │
              │  │ collision_prob     │  │  → 碰撞检测 (DetRate/FalseAlm)
              │  └───────────────────┘  │
              │                         │
              │  ┌── Geometric Mod ──┐  │
              │  │ visibility-aware  │  │
              │  │ approach-speed    │  │  → 路点修正 (WPColl%)
              │  │ multi-agent repul │  │
              │  └───────────────────┘  │
              └────────────┬────────────┘
                           │
                     safe waypoints (10×2)
                           │
                       Controller
```

**关键设计决策：检测与修正解耦**
- 碰撞检测由 V1 网络负责 → 高检测率、低误报率
- 路点修正由几何方法负责 → 最低路点碰撞率
- 两者独立运作，互不干扰

### 2.2 碰撞预测网络 V1 (CollisionPredictionNetwork)

**文件**: `coop_safety/learned/collision_network.py`
**参数量**: 46,659 (~191KB)
**功能**: 场景级碰撞概率预测

```
输入: agents (B, 30, 10)  每个 agent: [x, y, vx, vy, heading, length, width, speed, visible, type]
      mask (B, 30)         有效 agent 标记

AgentEncoderV2 (4,864 params):
  Linear(10 → 64) + ReLU + Linear(64 → 64) + ReLU
  输出: (B, 30, 64) per-agent embeddings

InteractionModule (16,768 params):
  MultiheadAttention(embed_dim=64, num_heads=4, batch_first=True)
  LayerNorm + 残差连接
  建模 agent 间多体交互 (如: A减速 → B不会撞C)
  输出: (B, 30, 64) interaction-aware embeddings

AttentionPool (8,385 params):
  Linear(64 → 1) → softmax → 加权求和 → Linear(64 → 128)
  学习"哪些 agent 最重要"的注意力权重
  输出: (B, 128) scene vector

CollisionHead (8,321 params):
  Linear(128 → 64) + ReLU + Linear(64 → 1) + Sigmoid
  输出: P(collision within 30 frames) ∈ [0,1]

TTCHead (8,321 params):
  Linear(128 → 64) + ReLU + Linear(64 → 1) + ReLU
  输出: estimated TTC (seconds, ≥0)
```

**训练配置**:

| 项 | 值 |
|----|-----|
| 数据 | DeepAccident val (104 scenarios, 8193 frames) |
| 正样本定义 | accident 场景中距碰撞 ≤30 帧 (12.7%) |
| Loss | Focal BCE (γ=2.0, α=0.75) + 0.1 × SmoothL1(TTC) |
| Optimizer | Adam, lr=5e-4, weight_decay=1e-5 |
| Epochs | 80, batch_size=64, 正样本过采样 |
| 结果 | **AUC=0.985, P=0.885, R=0.954, F1=0.918** |

### 2.3 改进碰撞预测网络 V2 (CollisionPredictionNetV2)

**文件**: `coop_safety/learned/collision_network_v2.py`
**参数量**: 66,291 (~260KB)
**功能**: 碰撞预测 + TTC回归 + 逐路点风险评分 (multi-task)

**vs V1 的创新**:

| 创新 | V1 | V2 |
|------|----|----|
| Agent 编码 | 对称 MLP | 可见性感知编码 (Embedding(2,16)) |
| 注意力机制 | 对称 Self-Attention | Ego-Centric Attention (ego→agents 不对称) |
| 接近速度 | 无 | approach_speed = -(pos·rel_vel)/dist |
| 路点评估 | 无 | WaypointRiskScorer (per-waypoint risk) |
| 多任务 | collision + TTC | collision + TTC + waypoint_risk |

```
输入:
  agents (B, 30, 12): [rel_x, rel_y, rel_vx, rel_vy, heading, length, width,
                        speed, visible(0/1), type_id, approach_speed, dist_to_ego]
  ego (B, 6): [speed, yaw_rate, ax, ay, heading, 0]
  waypoints (B, 10, 2): 待评估的候选路点

RelativeAgentEncoder (10,512 params):
  spatial_mlp: Linear(8→64) + GELU + Linear(64→64)    — 空间特征
  visibility_embed: Embedding(2, 16)                    — 可见/不可见分开编码
  type_embed: Embedding(6, 8)                           — 车型编码
  fuse: Linear(88→64) + GELU                            — 融合

EgoCentricAttention (25,536 params):
  ego_proj: Linear(6→64)                                — ego → query
  agent_k/v: Linear(64→64)                              — agents → key/value
  attn: MultiheadAttention(64, 4 heads)                 — ego 查询 agents
  agent_self_attn: MultiheadAttention                   — agent 间多体交互
  输出: scene_vector(64), agent_interacted(N,64), attention_weights(N)

SceneProjection (8,320 params):
  Linear(64→128) + GELU

CollisionHead (8,321 params):
  Linear(128→64) + GELU + Dropout(0.1) + Linear(64→1) → sigmoid

TTCHead (8,321 params):
  Linear(128→64) + GELU + Linear(64→1) → relu

WaypointRiskScorer (5,281 params):
  输入: [scene_vec(128); waypoint_xy(2)] = 130
  Linear(130→32) + GELU + Linear(32→32) + GELU + Linear(32→1) → sigmoid
  对每个路点独立输出风险分数 [0,1]
```

**训练配置**:

| 项 | 值 |
|----|-----|
| Loss | Focal BCE(γ=2.0,α=0.75) + 0.1×SmoothL1(TTC) + 0.2×BCE(wp_risk) |
| Optimizer | AdamW, lr=3e-4, weight_decay=1e-4 |
| Scheduler | CosineAnnealing, T_max=100 |
| Epochs | 100, batch_size=64, 正样本过采样(最多4×) |
| 路点风险标签 | 路点距 agent 预测位置 <3m → risk=1 |
| 结果 | **AUC=0.978, P=0.791, R=0.926, F1=0.854** |

### 2.4 Ours-Hybrid: Neural-Geometric 级联安全约束

**文件**: `coop_safety/learned/hybrid_safety.py`
**功能**: 结合 V1 检测 + 几何路点修正 + 多智能体排斥

#### 2.4.1 三个创新点

**创新1: 盲区感知安全缓冲 (Visibility-Aware Safety Buffer)**

不可见 agent (仅通过 V2X 协同感知检测到，ego 自身无法看到) 存在更高的位置不确定性。我们为不同可见性的 agent 分配不同安全边距：

$$
m_{\text{base}} = \begin{cases}
2.5 \text{m} & \text{if visible} \\
4.0 \text{m} & \text{if invisible (V2X only)}
\end{cases}
$$

**创新2: 接近速度自适应碰撞区 (Approach-Speed Adaptive Collision Zone)**

快速接近的 agent 威胁更大，安全边距随接近速度动态扩大：

$$
v_{\text{approach}} = -\frac{\mathbf{p} \cdot \mathbf{v}_{\text{rel}}}{|\mathbf{p}|}
$$

$$
m = m_{\text{base}} \times \left(1 + 0.3 \times \frac{\max(v_{\text{approach}}, 0)}{v_{\max}}\right) + 0.3 \times \max(l, w)
$$

其中 $\mathbf{p}$ 是 ego 到 agent 的相对位置，$\mathbf{v}_{\text{rel}}$ 是相对速度。

**创新3: 多智能体排斥场路点修正 (Multi-Agent Repulsive Field Correction)**

对比方法 (UniE2EV2X, MAP) 只处理最近的一个碰撞，可能将路点推向另一个 agent。我们对所有威胁 agent 累加排斥力：

$$
\mathbf{F}_i = \frac{\mathbf{p}_{\text{wp}} - \mathbf{p}_i}{|\mathbf{p}_{\text{wp}} - \mathbf{p}_i|^2} \times w_i
$$

$$
w_i = w_{\text{vis}} \times w_{\text{risk}} \times \frac{m_i - d_i + c}{d_i}
$$

其中 $w_{\text{vis}} = 1.0$ (可见) 或 $1.5$ (不可见)，$w_{\text{risk}} = 1 + r_{\text{wp}}$ (V2 路点风险分数)。

合成推移方向：

$$
\mathbf{d} = \frac{\sum_i \mathbf{F}_i}{|\sum_i \mathbf{F}_i|}, \quad \Delta = \max(\max_i(m_i - d_i + c), 1.0)
$$

$$
\mathbf{wp}_{\text{new}} = \mathbf{wp} + \mathbf{d} \times \Delta
$$

#### 2.4.2 完整流程

```
constrain_waypoints(waypoints, perception):

  1. V2 路点风险评分 (可选)
     → wp_risks[t] = V2.waypoint_risk(scene, wp[t])

  2. 几何碰撞检测 + 多智能体排斥修正 (始终执行)
     for t = 0..9:
       for each agent:
         predict position at dt = (t+1) × 0.5s
         compute visibility-aware margin
         if dist(wp[t], agent_pred) < margin:
           add to threat list
       if threats:
         wp[t] = multi_agent_repulsion(wp[t], threats, wp_risks[t])

  3. 轨迹平滑 (weight = 0.3)
     smoothed[t] = 0.7 × modified[t] + 0.3 × mid(t-1, t+1)

  4. 速度约束 (v_max = 20 m/s)
     clamp consecutive waypoint distance ≤ 10m

  5. V1 碰撞检测 (独立于路点修正)
     collision_prob = V1(agents, mask)
     is_dangerous = collision_prob > 0.3
     → 仅影响 DetRate/FalseAlm，不影响路点修正
```

#### 2.4.3 参数

| 参数 | 值 | 说明 |
|------|-----|------|
| base_margin_visible | 2.5m | 可见 agent 安全边距 |
| base_margin_invisible | 4.0m | 不可见 agent 安全边距 |
| approach_speed_factor | 0.3 | 接近速度缩放系数 |
| push_clearance | 1.0m | 碰撞解除后额外间距 |
| smooth_weight | 0.3 | 轨迹平滑权重 |
| v_max | 20 m/s | 速度上限 |
| detection_threshold | 0.3 | V1 碰撞检测阈值 |

### 2.5 三层规则安全约束 (Ours-Rule)

**文件**: `coop_safety/interface.py` → `SafetyConstraintModule`
**功能**: 纯规则驱动的三层安全约束（无神经网络）

```
constrain(perception) → SafeActionSpace:

  Step 1: 盲区推理
    identify_blind_spots(perception)  → 遮挡区域多边形
    infer_hidden_agents(blind_spots)  → 幽灵 agent

  Step 2: 三层风险评估
    RiskMap.build()     → 空间网格风险 (HIGH/MEDIUM/LOW)
    RiskGraph.build()   → 成对 TTC + 碰撞概率
    RiskEvents.enum()   → 碰撞事件列表 (类型/严重度/概率)

  Step 3: 初始可行域
    FeasibleRegion.compute()  → 可达集 ∩ 道路 - 碰撞区

  Step 4: 分层收紧
    HierarchicalConstraint.tighten()
      → RiskEvents 排除 → TTC 排除 → 风险区排除

  Step 5: 长远可行性检查
    FeasibilityCheck.check()  → 前瞻3秒模拟

  Step 6: (若不通过) 最小伤害模式
    MinimumHarmPlanner.plan()  → 12候选动作评估
```

**各模块参数**:

| 模块 | 关键参数 |
|------|---------|
| RiskMap | range=40m, cell=8m, HIGH≥0.35, MEDIUM≥0.15 |
| RiskGraph | prediction=5s, dt=0.1s, collision_radius=3m, max_ttc=8s |
| RiskEvents | min_prob=0.1, severity=0.4×speed+0.3×mass+0.3×angle |
| Hierarchical | ttc_critical=3s, ttc_warning=6s, buffer_scale=4.0 |
| FeasibilityCheck | horizon=3s, interval=1s, 4 action samples |
| MinHarm | 12 actions, sim_horizon=1.5s, dt=0.2s |
| BlindSpot | min_vehicle_length=3m, min_area=20m² |

## 三、对比方法

### 3.1 RSS [Shalev-Shwartz et al., 2017]

**出处**: "On a Formal Model of Safe and Scalable Self-driving Cars." arXiv:1708.06374

**算法**: 纵向安全距离公式：
$$d_{\text{safe}} = v_{\text{ego}} \rho + \frac{1}{2} a_{\max} \rho^2 + \frac{(v_{\text{ego}} + \rho a_{\max})^2}{2 a_{\min}} - \frac{v_{\text{other}}^2}{2 a_{\max,\text{brake}}}$$

参数: $\rho=1.0$s, $a_{\max}=3.0$, $a_{\min}=6.0$, $a_{\max,\text{brake}}=8.0$

**实现**: `experiments/methods.py` → `RSSOnly`

### 3.2 APF [Rasekhipour et al., IEEE T-ITS, 2017]

**出处**: "A Potential Field-Based Model Predictive Path-Planning Controller for Autonomous Road Vehicles"

**算法**: 各向异性排斥势场，沿行驶方向拉伸：
- $\sigma_{\text{forward}} = \sigma_{\text{base}} + \text{speed} \times 0.8$
- $\sigma_{\text{lateral}} = \sigma_{\text{base}} = 3.0$m
- 超过阈值 0.5 的区域排除

**实现**: `experiments/methods_modern.py` → `RiskPotentialField`

### 3.3 UniE2EV2X [Li et al., 2024]

**出处**: "Unified End-to-End V2X Cooperative Autonomous Driving" arXiv:2405.03971

**算法**: 几何多边形碰撞后处理
1. 对每个路点构建 ego oriented bounding box
2. 对每个 agent 预测位置构建 bbox
3. 若 bbox 距离 < 3.0m，将路点沿离开方向推移 2.0m
4. 只处理第一个碰撞

**实现**: `experiments/methods_new_baselines.py` → `UniE2EV2XSafety`

### 3.4 MAP [Yin et al., 2025]

**出处**: "MAP: End-to-End Autonomous Driving with Map-Assisted Planning" arXiv:2509.13926

**算法**: BBox 碰撞检测 + 最小位移修正
1. 构建 ego bbox + 0.5m clearance buffer
2. 检查与 agent bbox 的 intersection
3. 计算 overlap area → 沿离开方向推移 sqrt(overlap) + clearance
4. 只处理第一个碰撞

**实现**: `experiments/methods_new_baselines.py` → `MAPSafety`

### 3.5 RiskMM [Lei et al., 2025]

**出处**: "Risk Map As Middleware: Towards Interpretable Cooperative End-to-end Autonomous Driving" arXiv:2508.07686

**算法**: 高斯风险场 + 简化 MPC
1. 每个 agent 建立高斯风险场: $R = \exp(-d^2/(2\sigma^2))$, $\sigma = \max(l,3) + v \times 0.5$
2. 5次迭代梯度下降优化路点
3. 加平滑约束: 拉向前后路点中点
4. 速度约束: 相邻路点距离 ≤ $v_{\max} \times 0.5$

**实现**: `experiments/methods_new_baselines.py` → `RiskMMSafety`

## 四、数据集与评测

### 4.1 数据集

| 项 | 值 |
|----|-----|
| 名称 | DeepAccident (Wang et al., arXiv:2304.01168, 2023) |
| 类型 | CARLA V2X 仿真，含碰撞/正常配对场景 |
| 场景 | 104 (52 accident + 52 normal) |
| 帧数 | 8193 |
| 场景设计 | accident/normal 为同一交叉口的不同行为版本 |
| GT 感知 | 3D bbox (position/size/velocity/visibility) |
| 协同感知 | other_vehicle 视角的 GT 标注 |
| 加载器 | `experiments/deepaccident_loader.py` |

**场景配对关系**: accident 和 normal 共享相同的地图、交叉口、初始布局，但 agent 行为不同。Accident 版本最终发生碰撞，normal 版本安全通过。

### 4.2 评价指标

| 指标 | 计算方式 | 含义 |
|------|---------|------|
| **Detection Rate** | 碰撞前触发预警的 accident 场景数 / 52 | 能否发现碰撞 |
| **Early Warning** | 首次预警到 scenario 结束的帧数差 | 预警多早 |
| **False Alarm Rate** | normal 场景中误触发的场景数 / 52 | 是否过度敏感 |
| **Waypoint Collision Rate** | 路点距 agent <2m 的比例 / (帧数 × 10) | 路点安全性 |
| **Modification Rate** | 被修改的帧数 / 总帧数 | 介入频率 |

### 4.3 实验流程

```
对每个 scenario (104个):
  对每帧 (50-100帧):
    1. 加载 GT 感知 (DeepAccidentLoader)
    2. 生成 baseline waypoints (constant-velocity, 10路点×0.5s)
    3. 对每个方法:
       a. 传入 waypoints + 感知 → 安全约束
       b. 检查约束/修正结果
       c. 检查修正后 waypoints 的碰撞率
    4. 记录所有指标
```

### 4.4 评测结果 (10 methods × 104 scenarios, 5461s)

| 方法 | 碰撞检测率 | 提前预警帧 | 误报率 | 路点碰撞率 | 修改率 |
|------|-----------|-----------|--------|----------|--------|

| RSS [Shalev-Shwartz17] | 0.0% | 0.0 | 0.0% | 2.9% | 0.0% |
| APF [Rasekhipour17] | 0.0% | 0.0 | 0.0% | 2.9% | 0.0% |
| UniE2EV2X [Li24] | 100.0% | 57.2 | 100.0% | 0.7% | 65.9% |
| MAP [Yin25] | 96.2% | 42.3 | 94.2% | 0.4% | 29.6% |
| RiskMM [Lei25] | 100.0% | 60.0 | 100.0% | 2.7% | 98.6% |
| Ours-Rule | 100.0% | 59.9 | 100.0% | 2.9% | 92.3% |
| Ours-CollisionV1 | 100.0% | 27.8 | 26.9% | 2.9% | 16.7% |

### 4.5 结果分析

**Ours-Hybrid 全面领先**：

| 对比 | DetRate | WPColl% | FalseAlm |
|------|---------|---------|----------|
| vs UniE2EV2X | 同 100% | 0.2 vs 0.7 (↓71%) | 26.9% vs 100% (↓73%) |
| vs MAP | 100% vs 96.2% | 0.2 vs 0.4 (↓50%) | 26.9% vs 94.2% (↓71%) |

| 配置 | DetRate | WPColl% | FalseAlm | 说明 |

| 几何 only (无V1) | 98.1% | 0.2% | ~100% | 路点好但无法精确检测 |
| **V1 + 几何 (Hybrid)** | **100%** | **0.2%** | **26.9%** | 检测与修正解耦的最优 |

### 7.1 实验设计

使用 SUMO 微观交通仿真器生成极端、易碰撞的道路场景，评估安全约束方法在高风险环境下的表现。

### 7.7 闭环二次碰撞分析 (Closed-Loop)

**实验设计**: 每个方法独立控制 ego 车辆在 SUMO 中的行为（安全约束触发时通过 `traci.vehicle.slowDown` 减速 ego）。每个场景分别跑 7 次（7个方法），记录各方法导致的碰撞次数和二次碰撞率。

**二次碰撞定义**: N 次碰撞产生 N-1 次二次碰撞（第一次为初次碰撞，之后每次都算二次）。

#### 总体结果 (40 scenarios × 7 methods)

| 方法 | Ego碰撞 | 二次碰撞 | **二次碰撞率** | 平均严重度 | WPColl% |
|------|---------|---------|-------------|----------|---------|
| NoConstraint | 4 | 1 | **25%** | 12.6 m/s | 0.2% |
| RSS | 4 | 1 | 25% | 12.6 m/s | 0.2% |
| APF | 4 | 1 | 25% | 12.6 m/s | 0.2% |
| RiskMM | 4 | 1 | 25% | 12.6 m/s | 0.2% |
| UniE2EV2X | 21 | 18 | **86%** | 9.3 m/s | 0.0% |
| **Ours-Hybrid** | **27** | **24** | **89%** | **8.9 m/s** | **0.0%** |
| MAP | 47 | 44 | **94%** | 5.9 m/s | 0.0% |

#### 分场景详情 (Crossroads — 最危险场景)

| 方法 | Ego碰撞 | 二次碰撞 | 二次率 | 严重度 |
|------|---------|---------|--------|--------|
| NoConstraint | 2 | 1 | 50% | 13.6 m/s |
| RSS/APF/RiskMM | 2 | 1 | 50% | 13.6 m/s |
| UniE2EV2X | 16 | 15 | 94% | 8.4 m/s |
| **Ours-Hybrid** | **22** | **21** | **95%** | **8.1 m/s** |
| MAP | 41 | 40 | 98% | 5.2 m/s |

#### 关键发现与分析

**1. 介入型方法导致更多碰撞（但严重度降低）**

| 特征 | 不介入 (NoConstraint/RSS) | 介入 (Ours-Hybrid/MAP) |
|------|------------------------|---------------------|
| 碰撞总数 | 少 (4次) | 多 (27-47次) |
| 单次严重度 | **高** (12.6 m/s) | **低** (5.9-8.9 m/s) |
| 二次碰撞率 | 25% | 89-94% |
| 碰撞原因 | 正面冲突 (高速) | 被追撞 (低速) |

**原因**: 安全约束触发减速 → ego 突然变慢 → 后方车辆因 aggressive 驾驶参数（minGap=0.3m, tau=0.3s）来不及反应 → 追撞 → 连环碰撞。

**2. 虽然碰撞次数增加，但严重度大幅降低**

- NoConstraint: 4次碰撞 × 12.6 m/s = **50.4** 总伤害
- Ours-Hybrid: 27次碰撞 × 8.9 m/s = **240.3** 总伤害（表面更差）
- 但二次碰撞是低速追撞 (8.1 m/s)，vs 无约束时的正面高速冲突 (13.6 m/s)

**3. 启示: 闭环安全约束需要更精细的集成**

简单的"检测到危险→减速"策略在 aggressive 交通中会引发连环碰撞。更好的策略：
- 不仅减速，还需要**预判后方车辆**行为
- 温和制动而非急停（考虑减速度上限）
- 或者通过**变道**而非减速来避让

**这是未来工作的重要方向**: planning-level 的安全集成（修改轨迹规划而非简单覆盖速度控制）。
