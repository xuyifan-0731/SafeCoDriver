## 一、训练

### 1.1 数据集

| 项 | 值 |
|----|-----|
| 数据集 | DeepAccident (Wang et al., arXiv:2304.01168, 2023) |
| 类型 | CARLA 仿真 V2X 协同自驾，含碰撞和正常场景 |
| 数据路径 | `/raid/xuyifan/jiqiuyu/data/DeepAccident/` |
| 场景数 | 104 (accident 52 + normal 52) |
| 总帧数 | 8193 |
| 场景类型 | type1_subtype1 (27+27), type1_subtype2 (25+25) |
| 帧率 | ~20 fps |
| 碰撞标注来源 | meta 文件最后一个字段 = 碰撞帧号 (accident) / 总帧数 (normal) |
| GT 感知 | 每帧所有 agent 的 3D 框 (x,y,z,l,w,h,yaw,vx,vy,id,tick,visible) |
| 可见性 | `visible` 字段标记是否在 ego 视野内（True/False，体现盲区） |
| 协同感知 | `other_vehicle/label/` 提供另一辆车视角的 GT |
| 加载器 | `experiments/deepaccident_loader.py` → `DeepAccidentLoader` |

### 1.2 标签定义

| 标签 | 定义 |
|------|------|
| **正样本** (is_dangerous=1) | accident 场景中距碰撞帧 ≤30 帧的帧 (约 1.5s) |
| **负样本** (is_dangerous=0) | normal 场景全部帧 + accident 场景中远离碰撞的帧 |
| **TTC 标签** | frames_to_collision × 0.05s；安全帧设为 10.0s |

**标签分布**:
- 正样本: 1040 帧 (12.7%)
- 负样本: 7153 帧 (87.3%)
- 训练集: 6554 帧 (80%), 验证集: 1639 帧 (20%), shuffle 后分割

### 1.3 模型

| 项 | 值 |
|----|-----|
| 名称 | CollisionPredictionNetwork |
| 总参数 | 46,659 (~191KB) |
| 文件 | `coop_safety/learned/collision_network.py` |
| 模型权重 | `models/collision_net_best.pt` (191KB) |

**架构**:

```
AgentEncoderV2 (4,864 params):
  输入: (B, N, 10) — [x, y, vx, vy, heading, length, width, speed, visible, 0]
  Linear(10→64) + ReLU + Linear(64→64) + ReLU
  输出: (B, N, 64) per-agent embeddings

InteractionModule (16,768 params):
  Multi-head Self-Attention (4 heads, dim=64, batch_first)
  LayerNorm + residual connection
  捕捉 agent 间的多体交互 (A减速→B不会撞C)
  输出: (B, N, 64) interaction-aware embeddings

AttentionPool (8,385 params):
  Attention-weighted pooling over agents
  Linear(64→1) → softmax → weighted sum → Linear(64→128)
  输出: (B, 128) scene vector

CollisionHead (8,321 params):
  Linear(128→64) + ReLU + Linear(64→1) + Sigmoid
  输出: P(collision within 30 frames)

TTCHead (8,321 params):
  Linear(128→64) + ReLU + Linear(64→1) + ReLU
  输出: estimated TTC (seconds)
```

### 1.4 训练配置

| 项 | 值 |
|----|-----|
| 脚本 | `coop_safety/learned/train_collision.py` |
| Optimizer | Adam, lr=5e-4, weight_decay=1e-5 |
| Scheduler | CosineAnnealingLR, T_max=80 |
| Epochs | 80 |
| Batch size | 64 |
| Loss | Focal BCE (γ=2.0, α=0.75) + 0.1 × SmoothL1(TTC) |
| 正样本过采样 | 随机重复正样本至接近 1:1 平衡 |
| 梯度裁剪 | max_norm=1.0 |
| GPU | 1 × RTX 4090 |
| 训练耗时 | ~2 分钟 |

**Focal Loss 选择原因**: 标签不平衡 (12.7% 正样本)，标准 BCE 会偏向预测"安全"。Focal loss 的 γ=2.0 让模型更关注难分样本（接近碰撞但不确定的帧）。

### 1.5 训练结果

| 指标 | 值 |
|------|-----|
| Best AUC | **0.9847** (epoch 46) |
| Best Precision | 0.885 |
| Best Recall | 0.954 |
| Best F1 | 0.918 |

**训练曲线关键节点**:

## 二、测试对比

### 2.1 对比方法

| 方法 | 类型 | 出处 | 实现 |
|------|------|------|------|
| **Ours-Rule** | 规则三层安全约束 | 本文 | `coop_safety/interface.py` → `SafetyConstraintModule` |
| **Ours-Learned** (旧) | 学习型 RiskMap+RiskGraph (规则蒸馏) | 本文 | `coop_safety/learned/risk_assessor.py` |
| **Ours-Collision** (新) | 碰撞预测网络 (DeepAccident 训练) | 本文 | `coop_safety/learned/collision_network.py` |
| **RSS** | 纵向/横向安全距离 | Shalev-Shwartz et al., 2017 | `experiments/methods.py` → `RSSOnly` |
| **RPF** | 各向异性风险势场 | Rasekhipour et al., IEEE T-ITS 2017 | `experiments/methods_modern.py` → `RiskPotentialField` |
| **NoConstraint** | 无约束 | — | `experiments/methods.py` → `NoConstraint` |

### 2.2 测试数据

| 项 | 值 |
|----|-----|
| 数据集 | DeepAccident val (同训练集，但用验证集划分) |
| 场景 | 104 scenarios × ~80 frames |
| GT 感知 | 直接用 label 中的 3D 框（不经过检测模型） |
| 盲区 | `visible=False` 的 agent 作为不可见 agent（confidence=0.3）|
| 协同 | 可选加载 `other_vehicle/label/` 补全盲区 |

### 2.3 评价指标

#### 碰撞检测指标（DeepAccident 碰撞场景）

| 指标 | 定义 |
|------|------|
| **Collision Detection Rate** | accident 场景中，安全约束在碰撞前触发 CONSERVATIVE/MINIMUM_HARM 的比例 |
| **Early Warning Frames** | 首次触发预警距碰撞帧的帧数（越大=预警越早） |
| **Min-Harm Trigger Rate** | 触发 MINIMUM_HARM 模式的 accident 场景比例 |
| **False Alarm Rate** | normal 场景中触发约束的比例（越低越好） |

#### 碰撞预测指标（CollisionPredictionNetwork 输出）

| 指标 | 定义 |
|------|------|
| **AUC** | ROC 曲线下面积 |
| **Precision** | 预测为碰撞且确实碰撞的比例 |
| **Recall** | 实际碰撞中被正确预测的比例 |
| **F1** | Precision 和 Recall 的调和平均 |

#### 安全约束指标（所有场景）

| 指标 | 定义 |
|------|------|
| **Constraint Ratio** | 可行域面积被约束的比例 |
| **Min TTC** | 最小碰撞时间（安全余量） |
| **Avg Constraint Ratio** | 所有帧的平均约束率 |

#### DeepAccident 统一评测结果 (Ours-Collision v1)

以下为 Ours-Collision (v1) 在统一评测中的结果 (8 methods × 104 scenarios, 6181s):

| 方法 | 碰撞检测率 | 提前预警帧 | 误报率 | 路点碰撞率 | 修改率 |
|------|-----------|-----------|--------|----------|--------|
| NoConstraint | 0.0% | 0.0 | 0.0% | 2.9% | 0.0% |
| RSS [Shalev-Shwartz17] | 0.0% | 0.0 | 0.0% | 2.9% | 0.0% |
| APF [Rasekhipour17] | 0.0% | 0.0 | 0.0% | 2.9% | 0.0% |
| UniE2EV2X [Li24] | 100.0% | 57.2 | 100.0% | 0.7% | 65.9% |
| MAP [Yin25] | 96.2% | 42.3 | 94.2% | 0.4% | 29.6% |
| RiskMM [Lei25] | 100.0% | 60.0 | 100.0% | 2.7% | 98.6% |
| Ours-Rule | 100.0% | 59.9 | 100.0% | 2.9% | 92.3% |
| **Ours-Collision (v1)** | **100.0%** | 27.8 | **26.9%** | 2.9% | 16.7% |

**分析**:
- Ours-Collision(v1) 在误报率上优势明显 (26.9% vs UniE2EV2X 100%、Ours-Rule 100%)
- 检测率保持 100%
- 但路点碰撞率 2.9% = NoConstraint，因为 v1 只输出约束模式不修改路点
- 提前预警帧 27.8 低于 UniE2EV2X (57.2) 和 RiskMM (60.0)
- → 需要改进：(1) 增加路点修改能力 (2) 提高提前预警帧
